# PRD — 협업의 기술 발표 패키지

## 1) 배경
원문 문서(`협업의_기술_The_Art_of_Collaboration.md`)를 기반으로, 교육/멘토링 현장에서 바로 사용할 수 있는 발표 산출물을 일관된 구조로 관리한다.

## 2) 목표
- 원문 기반 PPTX/대본/발표자노트를 한 폴더에서 재현 가능하게 유지
- 생성 과정(스크립트)과 결과물(문서/슬라이드/에셋)을 분리 저장
- 30분/60분 발표 운영이 가능한 텍스트 자산 확보

## 3) 범위
### 포함
- 원문 md
- 기본/개선/확장 PPTX
- 30분/60분 발표 대본
- 슬라이드별 발표자 노트
- 이미지 생성/슬라이드 생성 스크립트
- Nanabanana 스타일 이미지 에셋

### 제외
- 외부 배포 자동화
- 영상 렌더링 자동화
- 발표용 웹앱 제작

## 4) 산출물 구조
- `assets/`: 시각 에셋(PNG)
- `scripts/`: 생성 스크립트(JS/TS)
- `docs/`: 부가 문서
- 루트: 원문 md, PPTX, 발표 대본 md

## 5) 성공 기준
- 발표자가 폴더 하나만 열어 30분/60분 발표를 수행 가능
- 슬라이드/대본/노트 간 메시지 불일치가 없음
- 재생성 스크립트로 핵심 PPTX를 다시 만들 수 있음

## 6) 리스크 및 대응
- 리스크: 로컬 환경별 의존성 차이(soffice, node 모듈)
- 대응: 스크립트와 결과물을 함께 보관하고, 생성 전제조건을 docs에 명시

## 7) 다음 액션
1. `docs/runbook.md`에 재생성 절차 정리
2. 최종 발표 버전(30분/60분) 태깅
3. 필요 시 발표 스크립트 구어체/행사용 분기
