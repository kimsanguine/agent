## ✅ 요구사항 (Requirements)

### 1) Problem & Goal
- **Problem**: 한국 PM이 AI를 실무에 적용할 때, 한국어 기반의 구조화된 실전 가이드가 부족하다.
- **Goal-1**: `ccforpms.com` 핵심 내용을 한국 PM 맥락으로 재구성한 한글 가이드북 요구사항을 정의한다.
- **Goal-2**: Obsidian 기반 초안 → habix.ai 게시 가능한 구조로 확장 가능한 기준을 만든다.

### 2) 타겟 사용자
- **Primary**: 한국 스타트업/IT 조직의 PM/PO
- **Secondary**: PM 취업 준비생, 주니어 PM, AI 도입을 준비하는 프로덕트 리더

### 3) 기능 요구사항 (Functional Requirements)
- **FR-001 (Must)**: 가이드북은 Level 0~3 학습 구조(입문→기초→실무→고급 자동화)를 포함해야 한다.
- **FR-002 (Must)**: 각 레벨에는 실제 PM 업무 기반 실습 과제(예: PRD 작성, 리서치, 데이터 분석)가 포함되어야 한다.
- **FR-003 (Must)**: 원문 단순 번역이 아니라 한국 PM 실무 맥락(국내 사례/업무 흐름)으로 재해석되어야 한다.
- **FR-004 (Should)**: Claude Code뿐 아니라 ChatGPT/Gemini/Cursor/Notion AI 등 멀티툴 활용 가이드를 포함해야 한다.
- **FR-005 (Should)**: 템플릿/프롬프트/워크플로우 재사용 가능한 산출물(노션, 프롬프트, 자동화 예시)을 제공해야 한다.
- **FR-006 (Could)**: 커뮤니티/멤버십 연계를 위한 질의응답/업데이트 동선을 설계한다.

### 4) 비기능 요구사항 (Non-Functional Requirements)
- **NFR-001 (Clarity)**: 비전공 PM도 이해할 수 있는 한국어 설명과 단계별 안내를 제공해야 한다.
- **NFR-002 (Practicality)**: 모든 모듈은 "읽고 끝"이 아닌 즉시 실행 가능한 체크리스트/실습 중심이어야 한다.
- **NFR-003 (Consistency)**: 모듈 간 용어, 난이도, 산출물 형식이 일관되어야 한다.
- **NFR-004 (Scalability)**: Obsidian 문서 구조가 향후 habix.ai 웹 콘텐츠로 확장 가능해야 한다.

### 5) 범위 정의
- **In Scope (현재)**
  - 한글 가이드북 PRD 요구사항 정의
  - 모듈 구조/실습 구조/차별화 포인트 확정
  - 베타 검증 및 런칭 준비 기준 수립
- **Out of Scope (현재)**
  - habix.ai 최종 프로덕션 개발/배포
  - 모든 산업군별 심화 트랙 동시 제작

### 6) 완료 기준 (Acceptance Criteria)
- **AC-001**: Phase 1~4 실행 항목이 요구사항과 일관되게 연결된다.
- **AC-002**: 타겟 독자/차별화/배포형태가 명확히 문서화된다.
- **AC-003**: 각 레벨별 학습목표 + 실습 과제가 정의된다.
- **AC-004**: 베타 테스트(3~5명) 피드백 반영 루프가 계획에 포함된다.

- [ ]  타겟 독자 정의 — 주니어 PM? 시니어 PM? 예비 PM?
- [ ]  차별화 포인트 확정 — ccforpms vs 내 가이드북의 차이

- [ ]  기존 콘텐츠 인벤토리 (노션 요약, 에이전트, 워크플로우 등)
- [ ]  배포 형태 결정 — 웹사이트? 노션? PDF? 강의?

## Phase 2: 구조 설계 (1주)

- [ ]  Level 0: 시작하기 — AI 도구 선택, 설치, 기본 개념
- [ ]  Level 1: 기초 — 프롬프팅, 파일 작업, 맥락 관리
- [ ]  Level 2: 실무 적용 — PRD 작성, 데이터 분석, 리서치
- [ ]  Level 3: 고급 — 에이전트 구축, 자동화, 오케스트레이션
- [ ]  각 레벨별 실습 과제 설계

## Phase 3: 콘텐츠 제작 (4주)

- [ ]  Level 0 작성 — 툴 비교(ChatGPT/Claude/Gemini), 설치 가이드
- [ ]  Level 1 작성 — 프롬프트 엔지니어링 기초, 파일/맥락 관리
- [ ]  Level 2 작성 — PM 실무 케이스 (PRD, 유저 리서치, 데이터 분석)
- [ ]  Level 3 작성 — 100 Agents 프레임워크, 자동화 워크플로우
- [ ]  실습 템플릿/파일 준비 (노션, 프롬프트, 에이전트 설정)

## Phase 4: 검증 & 런칭 (2주)

- [ ]  베타 리더 모집 (3-5명 PM)
- [ ]  피드백 수집 및 개선
- [ ]  랜딩 페이지/사이트 구축
- [ ]  런칭 콘텐츠 준비 (트위터, 링크드인, 뉴스레터)
- [ ]  공개 런칭

- 한국어 + 한국 스타트업/기업 맥락 (당근, 토스 등 사례)
