# 3.0 Nano Banana - Setup

## 1) 목적
- Nano Banana 트랙을 **5~10분 안에 바로 실행 가능한 상태**로 만든다.
- 이미지 생성 실습을 단발성 체험이 아니라, PM 실무에서 재사용 가능한 **시각 자산 파이프라인**으로 시작한다.
- 이후 3.1/3.2 모듈에서 막히지 않도록 계정·결제·폴더·명령어·로그 기준을 선제 정렬한다.

## 2) 원문 핵심 요약 (ccforpms)
1. 준비물은 명확하다: **Claude Code + Google 계정 + Google Cloud billing 결제수단**이 필수다.
2. 학습 시작은 지정 폴더에서 `/start-3-1-1` 실행으로 진행되며, **폴더 위치가 틀리면 명령이 동작하지 않는다**.
3. 기본 작업 루트는 `~/Documents/claude-code-pm-course`이며, 자동 다운로드 실패 시 zip 수동 다운로드/압축해제로 대체 가능하다.
4. 실습 모델은 **Gemini 3 Pro Preview (Nano Banana Pro)** 기반이며, 단순 생성이 아니라 배치 생성·파일 관리·반복 개선까지 포함한다.
5. 비용 가이드는 이미지당 약 **$0.10**, 모듈 전체는 대략 **$5 이하** 수준으로 안내된다.
6. Mac에서 `curl/unzip` 계열 이슈가 나면 `xcode-select --install`이 우선 대응이고, Windows는 수동 다운로드가 안전한 대안이다.
7. 이 단계의 진짜 목표는 “첫 이미지 1장”이 아니라, 3.1(기초/스타일) → 3.2(PM 시각화)로 이어지는 **지속 가능한 생성 운영 루프**를 세팅하는 것이다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선**: 설치 설명보다 “첫 실행 성공”을 완료 기준으로 둔다.
- **P2 맥락 우선**: 계정/결제/폴더/명령어/출력경로를 함께 세팅해 품질 편차를 줄인다.
- **P6 자동화 운영 기반**: 다운로드-실행-출력-리뷰를 표준 절차로 고정해 반복 비용을 낮춘다.
- **P8 Connected Hub**: Claude Code(실행)와 생성 모델(결과)을 분리하지 않고 하나의 작업 루프로 연결한다.
- **P9 Words-to-Workflow**: `/start-3-1-1` 같은 자연어 명령을 실제 학습·생성 워크플로 트리거로 사용한다.
- 3.x 트랙은 Optional이지만, v2.4 관점에서 **콘텐츠/마케팅 자산 생산성**을 끌어올리는 실전 레버로 활용한다.

## 4) 실무 적용 시나리오
**상황: 신규 기능 소개용 시각 자료를 2일 안에 준비해야 하는 PM**
1. 오전 1시간 내 Nano Banana 환경을 세팅하고 첫 실행까지 완료한다.
2. 팀 공용 폴더 구조(`assets/`, `prompts/`, `outputs/`, `logs/`)를 생성한다.
3. 실습용 샘플 3장을 생성해 생성 속도·품질·비용을 확인한다.
4. 실패 케이스(인증 오류, 경로 오류, 압축 오류) 대응 규칙을 팀 노션에 기록한다.
5. 다음 모듈(3.1.1)로 이동해 실제 프로젝트 맥락 이미지를 만들 준비를 끝낸다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 로컬 환경에서 Nano Banana 실행 경로와 `.env`를 점검하고, `/start-3-1-1` 진입까지 팀 공통 온보딩 절차를 확정한다.
- **수~목 실행**: 샘플 생성 3건을 수행해 속도/품질/비용을 기록하고, 자동 설치 실패 시 수동 설치 플로우를 실제로 리허설한다.
- **금요일 검증**: 신규 팀원 1명이 가이드를 보고 20분 내 첫 생성에 성공하는지 온보딩 테스트를 실행한다.
- **Done 기준**: 설치-실행-출력-장애대응까지 한 번에 따라할 수 있는 운영 가이드와 로그 템플릿이 준비되어 있고 첫 생성 기록이 남아 있다.
- **리스크/완화**: 결제 미연결·키 오류·경로 혼선 리스크는 사전 점검표(계정/키/경로/권한)로 막고, 플랫폼별 설치 차이는 OS별 분기 가이드로 완화한다.

## 5) 실습/적용 체크리스트
- [ ] `~/Documents/claude-code-pm-course` 경로에 코스 파일을 준비했다.
- [ ] 해당 경로에서 Claude Code를 실행하고 `/start-3-1-1` 입력까지 확인했다.
- [ ] Gemini API key 및 billing 준비 여부를 사전 점검했다.
- [ ] `outputs/`와 실험 로그 파일(생성시간/비용/프롬프트/결과)을 만들었다.
- [ ] 자동 다운로드 실패 시 수동 설치 절차를 팀 문서로 정리했다.
- [ ] “첫 생성 성공”을 트랙 시작 조건으로 팀 합의했다.

## 6) 산출물
- Nano Banana 실행 준비 완료된 로컬 환경
- 설치/실행/장애대응 1페이지 운영 가이드
- 공용 폴더 구조 및 실험 로그 템플릿
- 첫 실행 확인 기록(명령어, 실행 시각, 결과 경로)
- 팀 온보딩 검증 결과표(신규 1인 기준 소요 시간, 막힘 지점, 수정 반영 내역)
