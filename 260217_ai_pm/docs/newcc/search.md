# Search

- Search 주제를 빠르게 이해하고 바로 써볼 수 있게 정리한 문서입니다.

**이번 문서 목표**

## 1) 목적
- 검색을 “찾기 기능”이 아니라 **근거 수집과 의사결정 검증 프로세스**로 표준화한다.
- 문서/코드/이슈 탐색 시간을 줄이고, 근거 없는 판단을 차단한다.
- 추후 `5.2`와 통합 가능한 최소 독립 범위를 유지한다.

## 2) 원문 핵심 요약 (ccforpms)
- 원문 Search는 Pagefind 기반 client-side 검색으로 코스 페이지를 빠르게 탐색하는 용도다.
- 검색 정확도를 높이기 위해 phrase 검색, 복수 키워드 조합, 결과 섹션 확인을 권장한다.
- 원하는 결과가 없으면 키워드 변경(동의어 재검색) 후 구조 탐색을 병행하는 방식을 제시한다.
- 검색 실패 시 Company Context, Getting Started 등 상위 허브 페이지로 역추적하도록 안내한다.
- 핵심 메시지는 “검색창 1회 입력”이 아니라 “질문 분해 + 키워드 반복 조정 + 섹션 단위 확인”이다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P2 Context-rich**: 검색 요청 전 문제 정의를 분해해 키워드 세트를 설계한다.
- **P4 속도-검증 균형**: 빠른 검색 후, 반드시 파일 경로/라인/출처를 남겨 검증 가능성을 확보한다.
- **P9 Words-to-Workflow**: 자연어 질문을 검색 쿼리 템플릿으로 표준화해 반복 사용한다.
- **FR-013 정합**: Global/Project 컨텍스트 라우팅 규칙에 맞춰 검색 레이어(문서/코드/운영 로그)를 분리한다.
- **Automation vs Augmentation**: 대량 검색/초기 분류는 자동화, 해석과 의사결정은 PM이 수행한다.

## 4) 실무 적용 시나리오
### 시나리오: 온보딩 전환 저하 원인 탐색
1. 질문을 3개로 분해한다.
   - 어디서 이탈하는가?
   - 누구에게서 크게 발생하는가?
   - 최근 변경과 상관관계가 있는가?
2. 검색 레이어를 분리한다.
   - 문서: PRD/실험노트/회고
   - 데이터: 퍼널 리포트/코호트
   - 운영: 이슈/배포 로그
3. 키워드 조합 템플릿으로 1차 검색 후, 근거를 경로와 함께 수집한다.
4. 근거 부족 항목은 재검색(동의어/기간/세그먼트 변경)한다.
5. 최종적으로 “근거 있음/가설”을 분리해 PRD 가설 섹션에 반영한다.

### 이번 주 완료형 실행 플랜
- **월~화:** 온보딩 전환 이슈를 문제/원인/영향 3축으로 분해하고 검색 쿼리 템플릿 10개를 만든다.
- **수~목:** 문서·데이터·운영 3개 레이어에서 1차 검색을 수행해 근거 로그를 최소 20건 적재한다.
- **금 검증:** 사실/가설/미확인 분류 정확도를 리뷰어 1인과 교차검토하고, 오탐 원인 상위 3개를 정리한다.
- **Done 기준:** 쿼리 라이브러리 1종, 근거 로그 20건+, 재검색 규칙표 1종, PRD 반영 항목 3건 이상이 확보된다.
- **리스크·완화:** 키워드 편향으로 미탐지가 발생할 수 있으므로 동의어 사전과 기간/세그먼트 재검색 규칙을 강제 적용한다.

## 5) 실습/적용 체크리스트
- [ ] 질문을 문제/원인/영향으로 분해한 뒤 검색을 시작했다.
- [ ] 파일명 검색과 내용 검색, 맥락 문서 검색을 분리해 수행했다.
- [ ] 검색 근거를 경로/라인/출처와 함께 기록했다.
- [ ] 재검색 규칙(동의어, 기간 변경, 세그먼트 변경)을 사전에 정의했다.
- [ ] 최종 결과에서 사실/가설/미확인을 구분했다.

## 6) 산출물
- 검색 쿼리 라이브러리(문제유형별 템플릿)
- 근거 수집 로그(경로/라인/요약/신뢰도)
- 검색 회고 노트(미탐지 원인/쿼리 개선안)
- 온보딩 전환 이슈 근거 맵(레이어별 핵심 증거와 미확인 공백)

## 통합/삭제 판단 기준
- **통합 권고 조건**
  - `5.2-level2-claude-code-hub.md` 내 탐색 절차와 중복률 ≥ 75%
  - 검색 전용 템플릿 사용률이 4주 평균 20% 미만
  - 별도 문서 없이도 팀 평균 검색 소요시간 변동이 ±10% 이내
- **독립 유지 조건**
  - 신규 팀원 온보딩에서 검색 템플릿 사용 시 정확도 향상(예: 오탐률 20%↓)이 확인됨
  - 비개발 직군이 검색 문서를 독립적으로 참조할 필요가 큼
- **삭제/흡수 의사결정 방식**
  - 월간 운영 리뷰 2회 연속 통합 권고 조건 충족 시 본 문서를 `5.2` 부록으로 흡수
