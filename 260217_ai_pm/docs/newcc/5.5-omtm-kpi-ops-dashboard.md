## 1) 목적
- OMTM(One Metric That Matters) 중심으로 팀 실행을 정렬하고, 활동 나열형 보고를 제거한다.
- Level1~4 산출물을 동일 KPI 체계로 연결해 운영 의사결정을 빠르게 만든다.
- 자동화/에이전트 도입 효과를 수치로 검증한다.

## 2) 원문 핵심 요약 (ccforpms)
- 원문 company-context는 Activation/Retention/Time-to-Value/Viral Coefficient 같은 핵심 지표를 의사결정 중심에 둔다.
- 원문 2.2는 임팩트 추정과 실험 해석에서 단일값이 아닌 시나리오 기반 판단을 권장한다.
- 원문 2.3은 전략 문서가 실행 메트릭과 연결되어야 한다는 점을 강조한다.
- 원문 4.x는 계획-구현-배포 루프를 반복하며 결과를 검증하는 실전 패턴을 제공한다.
- 즉, KPI 대시보드는 별도 리포트가 아니라 실행 루프의 일부여야 한다.

## 3) 이든 철학 반영 포인트 (v2.4)
### Dennis baseline vs 이든 확장
| 항목 | Dennis baseline | 이든 Level5(운영지표) |
|---|---|---|
| 상태보고 | 텍스트 중심 | OMTM + 보조지표 + 다음 액션 3종 |
| 지표 연결 | 작업 단위 분절 | Level1~4 산출물과 KPI 직접 매핑 |
| 자동 갱신 | 수동 집계 | n8n 기반 주기 집계 + Slack 보고 |
| 실패 감지 | 회고 시 인지 | 임계값 알림 기반 조기 감지 |

- **FR-020** 직접 반영: OMTM 중심 상태보고 및 우선순위 연동.
- **M-014~M-017** 추적: 리드타임 단축, 자동화 성공률, 파일럿 전환 속도.
- **P10**: Validation Gap 단축 KPI(가설 검증 리드타임, 주당 검증 가설 수)를 운영 대시보드에 포함.
- **P4/P6**: 속도와 검증 균형을 KPI 게이트로 강제.

## 4) 실무 적용 시나리오
### 시나리오: 주간 운영 리뷰
1. 분기 OMTM 1개를 확정한다.
   - 예: Activation Rate
2. 보조지표 4개를 묶는다.
   - Time-to-Value, Retention, PRD→티켓 리드타임, 자동화 성공률
3. 매주 월요일 n8n이 데이터 수집/집계 후 Slack에 자동 공유한다.
4. 리뷰 미팅에서 다음 항목을 고정 순서로 점검한다.
   - OMTM 변화
   - 원인 가설 2개
   - 즉시 실행 액션 3개
5. 금요일에 실행 결과를 업데이트하고 다음 주 우선순위를 재배치한다.

### 실패 패턴
- 실패1: 지표가 많아 핵심이 흐려짐
  - 대응: OMTM 1개 + 보조지표 3~5개 상한
- 실패2: 데이터는 있으나 액션이 없음
  - 대응: 모든 지표 카드에 “owner + next action + due” 필수
- 실패3: 수집 지연으로 의사결정 타이밍 상실
  - 대응: 자동 수집 실패 알림 + 수동 백업 폼
- 실패4: 수치 개선이 체감성과와 연결 안 됨
  - 대응: KPI와 사용자 영향(불편 감소/전환 개선) 동시 기록

### 이번 주 완료형 실행 플랜
- **월~화:** 분기 OMTM 1개와 보조지표 3~5개를 확정하고 지표 정의서(산식/주기/owner)를 작성한다.
- **수~목:** n8n 기반 자동 집계와 Slack 공유를 연결하고 경보 임계값 테스트를 1회 수행한다.
- **금 검증:** 주간 리뷰에서 OMTM 변화, 원인 가설 2개, 다음 액션 3개를 실제로 도출해 백로그에 반영한다.
- **Done 기준:** 대시보드 운영본, 임계값/경보 규칙, 주간 리뷰 노트, 백로그 반영 링크가 준비된다.
- **리스크·완화:** 지표 해석 불일치로 논쟁이 길어질 수 있으므로 지표 사전(정의/산식/제외 조건)을 리뷰 전에 합의한다.

## 5) 실습/적용 체크리스트
- [ ] 분기 OMTM 1개와 선택 근거를 정의했다.
- [ ] 보조지표 3~5개를 OMTM과 인과 관계로 연결했다.
- [ ] 지표별 임계값/경보 규칙/담당자를 지정했다.
- [ ] 주간 리포트 자동 생성 및 Slack 공유를 설정했다.
- [ ] 리뷰 결과를 다음 주 백로그 우선순위에 반영했다.

## 6) 산출물
- OMTM 운영 대시보드(지표/임계값/owner)
- 주간 KPI 리뷰 노트(변화/원인/액션)
- 자동 집계/알림 워크플로우 문서(n8n 링크 포함)
- 지표 사전 카드(정의/산식/데이터 소스/해석 주의점)
