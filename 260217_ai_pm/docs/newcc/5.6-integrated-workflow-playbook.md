## 1) 목적
- Level1~4를 개별 문서가 아닌 **하나의 운영 플레이북**으로 통합한다.
- 아이디어→문서→티켓→자동화→프로토타입→운영 개선까지 단일 실행 루프를 제공한다.
- 팀이 상황별로 어떤 도구 조합을 선택해야 하는지 즉시 판단할 수 있게 한다.

## 2) 원문 핵심 요약 (ccforpms)
- 원문 코스 전반은 학습보다 실행을 강조하며, 각 모듈이 실제 산출물 생성으로 끝나도록 설계되어 있다.
- 원문 2.1~2.3은 PRD/분석/전략의 고급 워크플로우를 제시하고, 다중 관점 검증을 통해 품질을 높인다.
- 원문 1.4/1.5는 병렬 에이전트와 역할 기반 서브에이전트를 활용한 분업 방식을 보여준다.
- 원문 4.x는 계획→구현→반복→GitHub→배포로 이어지는 end-to-end 실행 루프를 제공한다.
- company-context 문서군은 모든 실행이 KPI/전략/사용자 맥락에 연결되어야 함을 전제로 한다.

## 3) 이든 철학 반영 포인트 (v2.4)
### Dennis baseline vs 이든 확장
| 항목 | Dennis baseline | 이든 통합 플레이북 |
|---|---|---|
| 워크플로우 구성 | 단계별 활용 | 단계 간 핸드오프 표준화 |
| 도구 사용 | 도구 중심 최적화 | 결과물 중심 도구 선택 |
| 운영 추적 | 개별 기록 | OMTM/KPI 기반 통합 추적 |
| 확장성 | 개인 효율 향상 | 팀 운영/자동화/에이전트 체계 확장 |

- **P8 Connected Hub**를 실무 플레이북 형태로 고정.
- **FR-016~FR-020**을 하나의 운영 루프에서 연결.
- **P10 Validation-First Discovery**: 단계별 실험을 병렬/단기 사이클로 운영해 “먼저 만들어보고 나중에 검증” 패턴을 방지.
- **P9 Words-to-Workflow**: 문장 명령 → 실행 산출물 → 운영 지표까지 단절 없는 흐름 확보.

## 4) 실무 적용 시나리오
### 통합 실행 순서 (표준 루프)
1. **문제 정의 (Claude.ai)**
   - 입력: company context + KPI
   - 산출물: 문제 정의 노트, 목표 지표
2. **실행 설계 (Claude Code / Level2)**
   - 입력: 문제 정의
   - 산출물: PRD 초안, 실험 설계, 명령어 실행 로그
3. **즉시 운영 (Level1)**
   - 실행: Notion 게시, 댓글 처리, Linear 티켓화, Slack 공유
   - 산출물: 실행 링크 세트
4. **자동화 확장 (Level3)**
   - 실행: n8n 스케줄/이벤트 플로우 연결
   - 산출물: 자동화 실행 로그, 실패 대응 기록
5. **고급 실행 (Level4)**
   - 실행: `/agent`로 prototype/eval/핸드오프
   - 산출물: 배포 후보 패키지
6. **성과 리뷰 (Level5)**
   - 실행: OMTM 변화 점검, 다음 주 우선순위 갱신
   - 산출물: 주간 운영 리포트

### 실패 패턴
- 실패1: 단계별 산출물 포맷이 달라 핸드오프 손실 발생
  - 대응: 공통 산출물 스키마(목표/근거/지표/owner/due) 강제
- 실패2: 도구만 바꾸고 운영 루프는 미변경
  - 대응: 도구 도입 KPI(리드타임/재작업률) 측정 의무화
- 실패3: 자동화 과신으로 검수 누락
  - 대응: 고위험 단계 human approval gate 유지
- 실패4: 회고 미실행으로 학습 축적 실패
  - 대응: 주간 회고를 `/today` 루틴에 고정 삽입

### 이번 주 완료형 실행 플랜
- **월~화:** 6단계 표준 루프의 공통 스키마(목표/근거/지표/owner/due)를 확정하고 팀에 배포한다.
- **수~목:** 실제 과제 1건을 선택해 Level1→Level4 구간을 축약 실행하며 단계별 핸드오프 누락을 점검한다.
- **금 검증:** OMTM 리뷰와 함께 루프 병목 2개 이상을 식별해 플레이북 개선 백로그로 등록한다.
- **Done 기준:** 통합 플레이북 v1, 핸드오프 템플릿 세트, 파일럿 실행 로그, 개선 백로그가 연결된다.
- **리스크·완화:** 단계 간 책임 공백이 생길 수 있으므로 각 핸드오프에 승인자와 승인 시점을 명시해 누락을 차단한다.

## 5) 실습/적용 체크리스트
- [ ] 아이디어→운영까지 6단계 표준 루프를 팀에 배포했다.
- [ ] 단계별 산출물 템플릿과 owner를 지정했다.
- [ ] Level1~4 실패 패턴 대응 규칙을 문서화했다.
- [ ] OMTM 기반 주간 리뷰 루틴을 캘린더에 고정했다.
- [ ] 플레이북 개선 백로그(월간)를 운영하기 시작했다.

## 6) 산출물
- 통합 운영 플레이북 v1 (단계/역할/지표 포함)
- 핸드오프 템플릿 세트(문서/티켓/자동화/에이전트)
- 플레이북 개선 로그(월간 업데이트 이력)
- 단계별 승인 게이트 표(승인자/승인조건/차단 이슈)
