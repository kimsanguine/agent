## 1) 목적
- AI를 PRD 대필기가 아닌 사고 파트너로 활용해, 빠르면서도 검증 가능한 PRD 초안을 만든다.
- 맥락 수집→질문 기반 정제→다중 관점 리뷰를 표준화해 고위험 문서의 재작업률을 낮춘다.

## 2) 원문 핵심 요약 (ccforpms)
- 모듈 시작은 `/start-2-1`이며, 목표는 PRD를 처음부터 혼자 쓰기보다 AI와 함께 사고를 구조화하는 것이다.
- `@-mentions`로 회사 문서, 사용자 리서치, PRD 템플릿, 방법론 문서를 함께 연결할수록 결과 구체성이 올라간다.
- Socratic questioning을 활용해 문제 정의, 해법 타당성, 성공 지표, 제약, 전략 적합성을 3~5개 질문으로 깊게 검증한다.
- 동일 문제에 대해 2~3개 전략안을 병렬 생성하면 첫 아이디어 고착을 줄이고 선택 품질을 높일 수 있다.
- sub-agent 다중 관점 리뷰가 핵심이다: Engineer(기술 리스크), Executive(사업 우선순위), User Researcher(사용자 적합성).
- 결과가 generic할 때는 컨텍스트 파일을 늘리고, 리뷰가 얕을 때는 agent 역할과 검토 범위를 더 구체화해야 한다.
- 최종 초안은 버전명을 체계화해 저장/공유하고 이후 반복 개선 루프에 투입한다.
- 고품질 PRD는 “AI가 대신 작성”보다 “PM이 질문과 비교 기준을 설계”할 때 나온다는 점이 강조된다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P2 맥락 우선 + Level 1~2**: `@`로 입력 컨텍스트를 충분히 연결해야 PRD 품질이 올라간다는 원문 원칙을 그대로 적용한다.
- **P3 PM 오케스트레이션 + Level 2**: PM이 질문 설계·대안 비교·리뷰 조율을 맡아 최종 의사결정 책임을 유지한다.
- **P4 속도-검증 균형 + Level 1**: 빠른 초안 생성 후 다중 관점 리뷰와 AC 점검으로 품질 게이트를 건다.
- **P7 Automation vs Augmentation + Level 1**: 초안 생성은 자동화, 최종 우선순위/범위 확정은 PM 직접 수행으로 역할을 분리한다.
- **P8 Connected Hub + Level 1~2**: PRD를 끝 문서로 두지 않고 Notion 게시/Linear 티켓화로 이어지는 실행 체인으로 확장한다.
- **P9 Words-to-Workflow + Level 4**: 질문·프롬프트·리뷰 지시문이 실제 실행 산출물(PRD 버전)로 직결되는 구조를 만든다.

## 4) 실무 적용 시나리오
시니어 PM이 신규 온보딩 기능 PRD를 2일 안에 제출해야 한다. 먼저 리서치/기존 정책/템플릿을 `@`로 묶어 초안을 만들고, Socratic 질문으로 리스크 가정을 점검한다. 이후 Engineer·Executive·User Researcher 리뷰를 병렬로 받아 상충 지점을 정리하고, 최종안은 AC와 KPI를 포함해 버전 태깅 후 공유한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: `/start-2-1`로 시작해 리서치, 기존 PRD, 정책, VOC 자료를 `@`로 4종 이상 연결하고 문제 정의/성공지표 초안(v0)을 작성한다.
- **수~목 실행**: Socratic 질문 5개로 핵심 가정을 검증하고 전략 대안 2~3개를 병렬 생성한 뒤 Engineer/Executive/User Researcher 리뷰를 반영해 v1을 만든다.
- **금요일 검증**: 문서 워크스루 1회를 진행해 AC, KPI, 제외 범위, 리스크 대응의 실행 가능성을 점검하고 최종본(vFinal)을 확정한다.
- **Done 기준**: PRD에 문제 정의, 목표 지표, AC, Out of Scope, 리스크/완화, 출시 후 검증 계획이 모두 포함되고 버전 이력까지 정리되어 있다.
- **리스크/완화**: 문서가 추상적으로 흐르면 사용자 원문 인용 3개와 로그 지표를 강제 포함하고, 리뷰 충돌 시 고객가치/매출영향/구현복잡도 기준표로 결론을 낸다.

## 5) 실습/적용 체크리스트
- [ ] `/start-2-1` 실행 후 입력 컨텍스트 파일 4종 이상을 `@`로 연결했다.
- [ ] Socratic 질문 5개(문제·해법·지표·제약·타이밍)를 작성했다.
- [ ] 대안 전략 2개 이상을 병렬 생성하고 비교 기준표를 만들었다.
- [ ] Engineer/Executive/User Researcher 리뷰를 반영해 수정 이력을 남겼다.
- [ ] 최종 PRD에 AC, KPI, 리스크 대응, 제외 범위를 명시했다.

## 6) 산출물
- PRD 초안 및 최종안(버전 기록 포함)
- Socratic 질문/응답 로그
- 다중 관점 리뷰 결과표
- 대안 비교표 및 의사결정 메모
- PRD 승인 회의용 1페이지 의사결정 요약(문제·대안·선택 근거·즉시 액션)
