## 1) 목적
- Claude Code를 PM 업무 자동화 도구가 아니라 `문제 정의-실행-검증` 운영 체계로 이해하게 한다.
- 입문 단계에서 “무엇을 자동화하고 무엇은 사람이 판단할지” 기준을 세워 초기 시행착오를 줄인다.

## 2) 원문 핵심 요약 (ccforpms)
- Claude Code는 로컬 파일/폴더를 직접 다루며 문서 작성, 분석, 정리를 대화 기반으로 실행한다.
- 핵심 기능 축이 구체적으로 제시된다: 파일 작업(`@`), 시각 분석, 병렬 에이전트, 커스텀 서브에이전트, `CLAUDE.md` 프로젝트 메모리.
- 사용 전제 조건이 명확하다: Claude Pro 또는 Max 구독, Mac/Windows/Linux 환경, 설치 약 15분.
- 학습 난이도 가이드가 제공된다: 총 10~12시간 학습으로 주당 5~10시간 업무 절감을 목표로 한다.
- 적용 영역은 PRD/로드맵, 고객 피드백 분석, 티켓 변환, 릴리스 노트, 이해관계자 커뮤니케이션, 경쟁/시장 리서치까지 확장된다.
- 일반 챗봇과의 차별점으로 지속 메모리, 실제 실행(파일/명령), 다문서 동시 처리, Git/MCP 연동이 강조된다.
- Monday.com PM의 Reddit 댓글 34,000건 분석 사례처럼 대규모 데이터에서도 PM 의사결정 속도를 높일 수 있음을 보여준다.
- 핵심 전환점은 “명령어 암기”가 아니라 “원하는 결과를 설계하고 오케스트레이션”하는 역량이라는 점이다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선 + Level 1**: 소개 단계부터 실제 산출물 생성 가능 범위를 제시해 학습 목표를 결과물 중심으로 고정한다.
- **P2 맥락 우선 + Level 2**: 품질이 프롬프트 길이보다 컨텍스트 구조에서 나온다는 원칙을 입문에서 먼저 심는다.
- **P3 PM 오케스트레이션 + Level 2**: PM을 작성자가 아닌 판단/우선순위/검증 루프 설계자로 정의해 역할 혼선을 줄인다.
- **P7 Automation vs Augmentation + Level 1**: 자동화 가능한 반복 작업과 PM 직접 판단이 필요한 영역을 구분해 과신을 막는다.
- **P8 Connected Hub + Level 1~3**: 단일 툴 학습이 아니라 MCP·협업도구·자동화로 이어지는 연결 관점을 초기부터 반영한다.

## 4) 실무 적용 시나리오
신입 PM이 첫 주에 기능 기획 문서를 써야 하는 상황에서, 0.0 문서를 기준으로 업무를 3단계로 나눈다. 1) 문제 정의와 성공 지표를 먼저 정리하고, 2) Claude Code로 기초 초안을 만들고, 3) PM이 최종 우선순위와 리스크를 확정한다. 이때 자동화는 자료 정리와 초안 생성에만 쓰고, 출시 판단은 사람이 맡도록 분리한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 현재 PM 업무를 `자동화/증강/직접판단` 3영역으로 나누고, 이번 주에 시험할 대표 업무 1개를 선정합니다.
- **수~목 실행**: 선정한 업무에 대해 Claude Code로 초안 생성→PM 검토→수정 반영까지 1회전 실행해 실제 리드타임을 측정합니다.
- **금요일 검증**: 실행 전후 시간을 비교해 절감 효과를 수치화하고, 품질 이슈(오해·누락·재작업) 빈도를 리뷰합니다.
- **Done 기준**: 팀이 공감하는 자동화 경계(어디까지 자동, 어디서부터 사람 판단)가 문서화되고 다음 주 재실행 항목이 확정됩니다.
- **리스크/완화**: 도구 과신으로 판단 책임이 흐려질 수 있으므로, 최종 승인 권한과 검토 체크포인트(정확성·비즈니스 적합성)를 PM에게 고정합니다.

## 5) 실습/적용 체크리스트
- [ ] 내가 맡은 PM 업무를 자동화 대상/증강 대상/직접 수행 대상으로 3분류했다.
- [ ] 다음 2주 안에 만들 핵심 산출물 1개(PRD/분석/전략)를 선택했다.
- [ ] 해당 산출물에 필요한 입력 컨텍스트(회사 문서, 사용자 리서치, 지표)를 목록화했다.
- [ ] Claude Code 적용 후 절감할 시간 목표(주간 기준)를 숫자로 설정했다.
- [ ] 실행 결과를 검증할 기준(정확성, 재작업률, 리드타임)을 정의했다.

## 6) 산출물
- PM 업무 3분류표(Automation/Augmentation/Manual)
- 2주 실행 목표 카드(산출물 1개 기준)
- 컨텍스트 입력 목록
- 성과 지표 초안(시간 절감·품질 기준)
- 주간 검증 리포트(전후 소요시간·오류율·재작업 사유)
