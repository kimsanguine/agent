# Home

**빠른 요약**
- NewCC 학습 흐름을 한눈에 잡는 시작 안내서입니다.

**이번 문서 목표**
- 설치부터 고급/자동화까지 개인·팀 학습 경로를 계획할 수 있습니다.

**처음 보는 분 가이드**
- 먼저 `## 1) 목적`과 `## 5) 실습/적용 체크리스트`만 읽고 시작하세요.
- 용어가 낯설면 [사전](./사전.md)을 먼저 확인하세요.

## 1) 목적
- NewCC 전체 학습 경로를 `설치 → 실행 → 기초 실무 → 고급 실무 → 자동화 확장` 순서로 정렬해, 처음 들어온 PM도 1주 안에 실행 루틴을 만들게 한다.
- 단순 문서 열람이 아니라, Claude.ai·Claude Code·n8n을 연결한 운영 모델(Level 1~4)로 학습 목표를 명확히 잡게 한다.

## 2) 원문 핵심 요약 (ccforpms)
- 이 코스는 영상 시청형이 아니라 Claude Code 안에서 과제를 직접 수행하는 인터랙티브 방식으로 설계되어 있다.
- 시작 표준 경로가 명확하다: 설치(0.1) → 저장소 준비(0.2) → `/start-1-1` 실행.
- 핵심 역량 축은 파일 작업(`@`), 병렬 에이전트, 커스텀 서브에이전트, `CLAUDE.md` 프로젝트 메모리, `@-mentions`, 이미지 분석이다.
- 실무 범위가 문서 작성에 한정되지 않는다. PRD 작성/리뷰, 데이터 분석, 경쟁 전략 수립까지 포함한다.
- 모듈 구조는 Module 0(준비), Module 1(기초), Module 2(고급 PM 실습)로 구성되어 단계적 난이도 상승이 가능하다.
- 작업 환경은 Nimbalyst·Obsidian·VS Code 중 선택 가능하며, 파일 변화의 실시간 확인이 권장된다.
- 코딩 경험이 없어도 자연어 지시로 학습 가능하게 설계되었고, 권장 학습 시간은 약 10~12시간이다.
- 과정은 Carl Vellotti 제작이며 Anthropic 공식 인증 코스가 아니라는 점이 명시되어 있다.

### Claude Code 핵심 기능 연결 (official 반영)
- Claude Code는 터미널 단독이 아니라 IDE/데스크톱 앱과 함께 쓰는 multi-surface 모델이므로, 학습 경로도 `어디서 실행하고 어디서 검토할지`까지 포함해 설계해야 한다.
- PM 반복 업무(요약, 정리, 비교, 체크리스트)는 CLI 자연어 지시로 자동화하고, 결과를 파일 단위로 즉시 축적해 운영 자산으로 전환할 수 있다.
- MCP를 연결하면 Slack/Notion/Linear 등 외부 맥락을 코스 실습 결과와 붙여서 `학습 → 실무 적용` 전환 시간을 줄일 수 있다.
- `CLAUDE.md`, skills, hooks를 팀 규칙으로 고정하면 온보딩 품질 편차를 줄이고 신규 PM도 동일한 산출물 형식을 재현할 수 있다.
- sub-agent/agent team 패턴으로 리서치·문서검토·요약을 병렬 처리하면 Module 학습 시간이 실제 실무 리드타임 단축으로 이어진다.
- git/PR 흐름과 결합하면 코스 산출물을 개인 메모가 아닌 팀 리뷰 가능한 운영 문서로 관리할 수 있다.
- CLI composability(`claude`, `/start-*`, `@파일`, 모드 전환)를 익히면 Module 0~5를 단일 실행 루프로 연결할 수 있다.

### ccforpms 단계별 실행 예시 (실전 재현)
1. **온보딩 목표를 5분 안에 고정한다.**
   - 실행 입력(프롬프트): `이번 주 NewCC 온보딩 목표를 3줄로 정리해줘. 필수 모듈만 포함하고 완료 기준도 넣어줘.`
   - 기대 출력: `0.x 설치 완료`, `1.x 핵심 기능 실습`, `2.x 첫 실무 산출물` 형태의 3줄 목표.
   - 체크포인트: 목표마다 완료 기준이 숫자/행동으로 적혀 있는지 확인.

2. **필수·선택 학습 경로를 분리한다.**
   - 실행 입력(프롬프트): `docs/newcc 기준으로 필수(0.x~2.x)와 선택(3.x~5.x) 모듈을 표로 정리해줘.`
   - 기대 출력: 모듈명/목적/산출물/예상시간이 포함된 표 1개.
   - 체크포인트: 필수 경로에 `/start-1-1`, `/start-1-3`, `/start-1-6`, `/start-2-1`이 포함되는지 확인.

3. **첫 실행 커맨드로 진입 상태를 검증한다.**
   - 실행 입력(명령): `claude "/start-1-1"`
   - 기대 출력: 1.1 Welcome 모듈 안내와 다음 액션 제안.
   - 체크포인트: 현재 작업 디렉터리가 코스 루트인지, 모듈 안내가 정상 표시되는지 확인.

4. **문서 기반 실행 체크리스트를 추출한다.**
   - 실행 입력(프롬프트): `@docs/newcc/1.3-fundamentals-first-tasks.md 읽고 이번 주 바로 실행할 항목 5개만 체크리스트로 뽑아줘.`
   - 기대 출력: 채널 변환, @파일 참조, output style 등 우선순위 체크리스트.
   - 체크포인트: 각 항목이 "무엇을/언제/어떻게 검증"으로 표현되는지 확인.

5. **팀 공유용 첫 산출물을 만든다.**
   - 실행 입력(프롬프트): `온보딩 진행현황을 팀장 공유용 8줄 브리핑으로 작성해줘. 리스크 2개와 대응 2개 포함.`
   - 기대 출력: 즉시 붙여넣기 가능한 브리핑 텍스트.
   - 체크포인트: 리스크가 추상어가 아닌 실제 장애(설치 실패, 경로 오류)로 적혔는지 확인.

6. **품질 게이트를 별도 요청으로 돌린다.**
   - 실행 입력(프롬프트): `지금까지 만든 온보딩 문서에서 검증 기준이 없는 항목만 찾아 수정안을 제시해줘.`
   - 기대 출력: 누락 항목 목록 + 보완 문장.
   - 체크포인트: "완료" 판단 문장이 전부 측정 가능 표현(예: 0건, 3회, 10분 이내)인지 확인.

7. **주간 실행 루틴으로 고정한다.**
   - 실행 입력(프롬프트): `다음 주에도 반복 가능한 온보딩 운영 루틴을 월~금 1페이지로 정리해줘.`
   - 기대 출력: 주간 운영표 초안.
   - 체크포인트: 담당자, 산출물 위치, 금요일 리뷰 포맷까지 포함되는지 확인.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선 + Level 1**: 홈에서 바로 실행 순서와 첫 명령(`/start-1-1`)을 고정해 “읽고 끝나는 가이드”를 방지한다.
- **P3 PM 오케스트레이션 + Level 2**: 학습자가 작성자보다 조율자 역할(요청 분해, 리뷰 루프 설계)을 먼저 익히도록 경로를 배치한다.
- **P8 Connected Hub + Level 1~4**: Claude.ai·Claude Code·n8n의 단계 확장을 홈에서 먼저 보여줘 도구 단절 사용을 줄인다.
- **P9 Words-to-Workflow + Level 4**: 자연어 지시가 실제 산출물/자동화로 이어진다는 관점을 코스 입구에서 명확히 심는다.
- **P5 한국 실무 적합성 + Level 2**: 원문 흐름을 유지하되 한국 PM의 주간 운영(브리핑-실행-회고) 문맥으로 재배치한다.

## 4) 실무 적용 시나리오
신규 PM 3명이 팀에 합류한 상황에서 Leader는 홈 문서를 온보딩 표준으로 사용한다. Day 1에는 0.x만 완료해 설치/실행 상태를 맞추고, Day 2~3에는 1.x로 파일 작업·에이전트 협업·메모리 규칙을 통일한다. Week 2부터 2.x 실습으로 PRD와 분석 산출물을 만들게 하고, 이후 Level 3~4 자동화 후보를 팀 공통 백로그로 넘긴다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 팀 온보딩 기준으로 `0.x→1.x→2.x` 필수 경로를 확정하고, 각 모듈 담당자와 완료 예정일을 캘린더에 등록합니다.
- **수~목 실행**: 개인별 필수 모듈 실습을 끝내고 모듈당 최소 1개 산출물(PRD 초안/분석 요약)을 `your-work`에 업로드합니다.
- **금요일 검증**: 30분 리뷰 세션에서 실행 기록과 산출물 품질(정확성, 재작업률)을 동시에 점검하고 다음 주 개선 항목을 합의합니다.
- **Done 기준**: 참여자 전원이 `/start-1-1`, `/start-1-3`, `/start-1-6`, `/start-2-1` 실행 로그와 2주 로드맵 초안을 제출합니다.
- **리스크/완화**: 모듈 난이도 편차로 일정이 밀리면 선택 경로(3.x~5.x)는 보류하고 필수 경로 완주를 우선하며, 숙련자-신규 1:1 페어로 막힘 구간을 해소합니다.

- **초보자 운영 팁 1**: 매일 시작 전 "오늘 완료 기준 1개"를 채팅 상단에 적고 시작합니다.
- **초보자 운영 팁 2**: 산출물은 로컬 임시 폴더가 아니라 팀이 합의한 `your-work` 하위 경로에만 저장합니다.
- **초보자 운영 팁 3**: 리뷰는 "좋다/별로"가 아니라 `정확성/재현성/재사용성` 3축으로만 평가합니다.
- **롤백/복구 액션**: 금요일 검증에서 2개 이상 모듈이 미완료면, 다음 주 월요일 오전을 `복구 슬롯`으로 고정하고 선택 모듈 학습을 전면 중단한 뒤 필수 모듈 미완료자 페어 실습으로 재진행합니다.

### 자주 막히는 지점과 해결 가이드
1. **막힘: 모듈은 실행했는데 산출물이 팀 폴더에 안 쌓임**
   - 원인: 개인별 임의 경로 저장, 파일명 규칙 불일치.
   - 해결: `your-work/이름-모듈명-날짜.md` 규칙을 고정하고, 하루 1회 경로 점검 체크를 진행합니다.

2. **막힘: 필수/선택 모듈 경계가 모호해 일정이 밀림**
   - 원인: 학습 욕심으로 3.x~5.x를 조기 착수.
   - 해결: 주간 목표를 `0.x~2.x 완주`로 제한하고, 선택 모듈은 금요일 Done 기준 달성 후에만 오픈합니다.

3. **막힘: 실행은 빠른데 품질 검증이 빠짐**
   - 원인: 산출물 제출만 하고 검증 기준 미기록.
   - 해결: 모든 문서 하단에 `검증 기준 3개`를 템플릿으로 붙여 의무화합니다.

4. **막힘: 팀원별 이해도가 달라 리뷰 시간이 과도하게 길어짐**
   - 원인: 공통 용어 부재, 모듈별 기대 산출물 불명확.
   - 해결: 주중 15분 용어 싱크 미팅을 운영하고, 모듈별 샘플 산출물 1개를 기준본으로 지정합니다.
