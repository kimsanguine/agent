## 1) 목적
- 시리즈형 이미지 제작에서 스타일 편차를 줄이고, 발표/제안/마케팅 자산의 **톤 일관성**을 확보한다.
- 재생성 난사를 줄이고, **선택안 정제(iteration)** 중심의 생산 체계를 정착시킨다.
- style·subject reference를 전략적으로 결합해 PM 산출물의 신뢰도를 높인다.

## 2) 원문 핵심 요약 (ccforpms)
1. Golden Rule: 처음부터 다시 생성하기보다 **기존 결과를 구체적으로 수정**하는 편이 품질/속도 모두 유리하다.
2. 태그 나열(tag soup)보다 자연어 브리프가 효과적이며, subject/setting/lighting/mood/textures/colors/composition을 명확히 써야 한다.
3. reference images는 목적별로 나눈다: **Style reference**(톤), **Subject reference**(인물/제품 동일성), **Composition reference**(배치).
4. subject 일관성은 보통 reference 3~5장일 때 안정성이 높다.
5. variants는 방향 탐색(2~3개), iteration은 선택안 완성으로 역할을 분리해야 한다.
6. grid 생성 시에는 크기(예: 3x3), 셀 내용, 공통 일관성 조건을 명시해야 품질이 올라간다.
7. 실패 패턴은 스타일 미스매치, 캐릭터 불일치, grid 불균일, 변형안 유사도 과다이며, 모두 지시 구체화로 완화 가능하다.
8. “meaningfully different creative interpretations” 같은 지시를 넣으면 variant 다양성을 높일 수 있다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P2 맥락 우선**: 스타일 일관성은 마법 키워드가 아니라 참조 품질과 사용 맥락 정의에서 나온다.
- **P4 속도-검증 균형**: variants로 빠르게 탐색하고 iteration으로 검증·수정해 낭비를 줄인다.
- **P5 한국 실무 적합성**: 캠페인/제안서/세일즈덱처럼 다장 구성물이 많은 국내 PM 환경에서 일관성 관리가 핵심이다.
- **P6 자동화 운영 기반**: 반복 지시 패턴을 템플릿화하면 팀 단위 제작 속도가 올라간다.
- **P7 Automation vs Augmentation**: 변형 생성은 자동화, 브랜드 톤 최종 판정은 PM/디자이너의 증강 판단으로 남긴다.
- **P9 Words-to-Workflow**: 자연어 브리프 품질이 곧 산출물 일관성 품질이라는 원칙을 실습으로 체득한다.

## 4) 실무 적용 시나리오
**상황: 동일 기능을 설명하는 5장 슬라이드 비주얼 제작**
1. 첫 장에서 스타일 기준(색감/질감/조명/카메라 톤)을 확정한다.
2. 나머지 4장은 subject와 메시지만 바꾸고 스타일 코어 문장은 고정한다.
3. 장별로 variants 2개씩 뽑아 전달력 높은 안을 고른다.
4. 선택안만 iteration으로 정리해 세트 전체의 균형을 맞춘다.
5. 최종본은 2K로 통일 출력해 문서/발표 채널에 재사용한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 스타일 코어 문장(색감, 질감, 조명, 구도)을 확정하고 style/subject/composition reference 세트를 최소 1개 프로젝트 기준으로 정리한다.
- **수~목 실행**: 장표 5장 분량을 variants 2개씩 생성한 뒤 선택안만 iteration으로 정리해 세트 전체 톤을 맞춘다.
- **금요일 검증**: 팀 리뷰에서 일관성 루브릭(색, 질감, 인물 동일성, 레이아웃)을 점수화해 편차가 큰 장표를 재작업한다.
- **Done 기준**: 최소 5장 이상의 시리즈 자산이 같은 스타일 코어를 유지하고, 편차 수정 이력과 최종 선택 근거가 문서로 남아 있다.
- **리스크/완화**: style drift 위험은 고정 문장과 참조 이미지 세트를 잠그는 방식으로 완화하고, variant 과다 생성으로 시간 낭비 시 라운드별 생성 수 상한을 둔다.

## 5) 실습/적용 체크리스트
- [ ] 스타일 코어 문장(고정 요소)을 정의했다.
- [ ] style/subject/composition reference를 목적별로 분리해 적용했다.
- [ ] variants(탐색)와 iteration(완성) 단계를 분리 운영했다.
- [ ] 동일 세트 5장 이상에서 일관성 루브릭(색/질감/구도)을 점검했다.
- [ ] 편차 원인을 프롬프트 단위로 기록하고 수정 규칙으로 환원했다.
- [ ] 최종본은 해상도/톤 규격을 통일해 내보냈다.

## 6) 산출물
- 스타일 코어 프롬프트(고정 요소 정의 포함)
- 시리즈 이미지 세트(최소 5장)
- 일관성 평가표(항목별 편차/수정 이력)
- variants→iteration 운영 로그
- 스타일 편차 수정 히스토리(원인 프롬프트, 수정 문장, 개선 결과 before/after)

  2. 실제 운영에서 스타일 관리는 3.1.4(스타일 DB) 단계에서 일괄 처리하는 편이 효율적이다.
  3. 코호트 피드백에서 “개념 중복 대비 실습 차별성 부족” 의견이 반복된다.
- 단, 브랜드 일관성 요구가 높은 조직(콘텐츠/마케팅 중심)에서는 독립 모듈 유지 가치가 높다.
