## 1) 목적
- 코스 저장소 준비부터 첫 모듈 실행까지를 한 번에 완료하는 부트스트랩 절차를 정착시킨다.
- 경로/컨텍스트 오류를 줄여 `/start-*` 명령이 정상 동작하는 환경을 확보한다.

## 2) 원문 핵심 요약 (ccforpms)
- 기본 목표는 PM이 복잡한 git 절차를 직접 암기하지 않고, Claude에게 작업 의도를 말해 실행하는 패턴을 익히는 것이다.
- 권장 기본 흐름이 제시된다: `~/Documents` 이동 → 코스 zip 다운로드 → `claude-code-course`로 압축 해제 → `claude "/start-1-1"` 실행.
- 표준 명령 체인이 구체적으로 제공된다: `cd`, `curl -L ... -o course.zip`, `unzip`, `claude` 순서.
- 대안 흐름도 존재한다: 먼저 `claude` 실행 후 다운로드/압축 해제를 요청하고, 폴더 재진입해 학습을 시작한다.
- 장애 대응 포인트가 명확하다: `curl`/`unzip` 미설치, 다운로드 실패, 저장 위치 변경 시 수동 다운로드로 우회 가능.
- 시작 후 검증 질문 예시가 제공된다: "Show me all the Level 1 modules", "List the learning objectives for Module 1.3".
- `/start` 명령이 정상 동작하려면 반드시 `claude-code-course` 폴더 내부 컨텍스트에서 실행해야 한다.

### Claude Code 핵심 기능 연결 (official 반영)
- 시작 단계부터 Claude Code는 터미널 실행 + 에디터 검증을 함께 쓰는 multi-surface 흐름이 효율적이며, 경로 오류를 빠르게 발견할 수 있다.
- 저장소 준비 자동화(`download/unzip/start`)는 PM 온보딩의 첫 반복 작업 자동화 사례로 활용하기 좋다.
- MCP 연동을 염두에 두면 초기에 폴더 구조/권한을 정리해 이후 외부 도구 연결 시 컨텍스트 누락을 줄일 수 있다.
- `CLAUDE.md`에 부트스트랩 규칙(표준 경로/명령 순서/검증 질문)을 고정하면 신규 입사자도 같은 실행 결과를 낼 수 있다.
- sub-agent/agent team 학습 전제 조건은 정확한 프로젝트 컨텍스트이므로, clone/start 단계의 경로 정확성이 병렬 품질을 좌우한다.
- git/PR 관점에서 코스 셋업 결과(체크리스트, 실패 로그)를 문서화해 팀 지식으로 축적할 수 있다.
- CLI composability(`cd`, `curl`, `unzip`, `claude "/start-*"`)를 하나의 체인으로 다루면 재현성과 교육 효율이 높아진다.

### ccforpms 단계별 실행 예시 (실전 재현)
1. **작업 루트를 고정한다.**
   - 실행 입력(명령): `cd ~/Documents`
   - 기대 출력: 문서 저장 루트 이동 완료.
   - 체크포인트: 개인별 임의 경로가 아닌 팀 표준 루트 사용 여부 확인.

2. **코스 압축 파일을 내려받는다.**
   - 실행 입력(명령): `curl -L <코스-URL> -o course.zip`
   - 기대 출력: `course.zip` 생성.
   - 체크포인트: 다운로드 실패 시 브라우저 수동 다운로드로 즉시 우회.

3. **압축 해제로 학습 폴더를 만든다.**
   - 실행 입력(명령): `unzip course.zip -d claude-code-course`
   - 기대 출력: `claude-code-course` 디렉터리 생성.
   - 체크포인트: 폴더명 오타/중복 폴더 중첩 여부 확인.

4. **코스 루트에서 Claude를 실행한다.**
   - 실행 입력(명령): `cd claude-code-course && claude "/start-1-1"`
   - 기대 출력: 1.1 시작 안내.
   - 체크포인트: 상위 폴더에서 실행하지 않았는지 경로 재확인.

5. **컨텍스트 로딩을 질문으로 검증한다.**
   - 실행 입력(프롬프트): `Show me all the Level 1 modules.`
   - 기대 출력: Level 1 모듈 목록.
   - 체크포인트: 목록이 누락 없이 표시되는지 확인.

6. **학습 목표를 추가로 검증한다.**
   - 실행 입력(프롬프트): `List the learning objectives for Module 1.3.`
   - 기대 출력: 1.3 목표 요약.
   - 체크포인트: 모듈 목표를 실제 업무로 번역 가능한지 확인.

7. **장애 대응 경로를 기록한다.**
   - 실행 입력(프롬프트): `다운로드/압축/경로 오류 대응표를 작성해줘. 실패 원인과 우회 절차 포함.`
   - 기대 출력: 온보딩 대응표 1부.
   - 체크포인트: 초보자도 10분 내 따라할 수 있는지 검토.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선 + Level 2**: 저장소 준비를 이론 설명 대신 즉시 실행 가능한 커맨드 체인으로 제공해 진입 시간을 단축한다.
- **P2 맥락 우선 + Level 2**: 올바른 작업 디렉토리에서 시작하도록 강제해 컨텍스트 누락으로 인한 품질 저하를 막는다.
- **P8 Connected Hub + Level 1**: 시작 단계부터 문서·명령·실행 결과를 연결해 이후 MCP/협업도구 연동으로 자연스럽게 확장한다.
- **P9 Words-to-Workflow + Level 2**: 자연어 요청으로 다운로드/정리/검증까지 이어지는 흐름을 체득하게 만든다.

## 4) 실무 적용 시나리오
신규 입사 PM이 첫날 과제로 코스 환경을 셋업해야 한다. 문서에 제시된 커맨드 체인을 그대로 실행해 `/start-1-1`까지 도달하고, 이어서 레벨/모듈 확인 질문으로 컨텍스트 정상 로딩을 검증한다. 실패 시 수동 다운로드로 우회하고, 어떤 단계에서 막혔는지 체크리스트에 기록해 다음 입사자 온보딩 자료로 축적한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 부트스트랩 표준 명령(`cd → curl → unzip → claude`)을 팀 공통으로 실행하고, 개인별 작업 경로를 동일한 루트로 맞춥니다.
- **수~목 실행**: `/start-1-1` 실행 후 모듈 조회 질문 2~3개로 컨텍스트 로딩을 검증하고 실패 케이스는 원인별로 분류합니다.
- **금요일 검증**: 신규 사용자 관점 리허설(처음부터 다시 셋업)을 진행해 문서만으로 재현 가능한지 점검합니다.
- **Done 기준**: 팀원 전원이 `claude-code-course` 내부에서 시작 명령 실행에 성공하고, 실패/우회 절차가 온보딩 문서에 반영됩니다.
- **리스크/완화**: 네트워크 제한이나 압축 해제 도구 부재로 초기 실패가 발생할 수 있어, 수동 다운로드 링크와 대체 압축 해제 도구 안내를 사전 제공합니다.

- **초보자 운영 팁 1**: 명령 실행 전 `pwd`로 현재 경로를 먼저 확인합니다.
- **초보자 운영 팁 2**: 압축 해제 후 폴더가 중첩되면 바로 정리하고 재시작합니다.
- **초보자 운영 팁 3**: 검증 질문 2개를 통과하기 전에는 다음 모듈로 넘어가지 않습니다.
- **롤백/복구 액션**: 경로가 꼬여 `/start-1-1`이 실패하면 기존 코스 폴더를 보존 백업한 뒤 새 폴더로 재압축 해제하고, 시작 명령을 루트에서 다시 실행합니다.

### 자주 막히는 지점과 해결 가이드
1. **막힘: 압축 해제 후 폴더 구조가 예상과 다름**
   - 해결: 중첩 폴더를 정리해 `claude-code-course` 루트 하나만 유지하고 재진입.

2. **막힘: `/start-1-1`이 동작하지 않음**
   - 해결: 현재 경로가 코스 루트인지 확인, 상위/하위 오진입이면 루트 재이동 후 재실행.

3. **막힘: 네트워크 문제로 `curl` 실패**
   - 해결: 브라우저 수동 다운로드로 전환하고 파일 무결성(크기/확장자) 확인 후 진행.

4. **막힘: 초보자가 실패 원인을 기록하지 않아 재발**
   - 해결: 실패 시점/오류문구/해결시간 3항목을 의무 기록하도록 체크리스트에 고정.

### Claude Code 핵심개념 체크리스트 (표준 템플릿)
- [ ] 이 문서의 목표를 **문제-행동-성과지표** 1문장으로 적었다.
- [ ] 작업 시작 전에 **모드(Plan / Edit / Auto-Accept)** 선택 기준을 기록했다.
- [ ] `CLAUDE.md`의 프로젝트 규칙(금지사항/품질게이트/보고형식)을 확인했다.
- [ ] `@` 또는 첨부 자료로 **맥락 입력(문서/데이터/정책) 3개 이상** 연결했다.
- [ ] 필요한 경우 **sub-agent/병렬 작업** 범위와 합성(synthesis) 책임자를 지정했다.
- [ ] **MCP/외부도구 연동 필요 여부**와 접근 권한 상태를 점검했다.
- [ ] 산출물에 **검증 기준(정확성/재현성/지표/테스트)** 을 명시했다.
- [ ] 결과를 **Git/문서 링크/로그**로 추적 가능하게 남겼다.
- [ ] 다음 액션 3개를 **담당자·기한·검증포인트**와 함께 확정했다.

## 5) 실습/적용 체크리스트
- [ ] 코스 파일을 `claude-code-course` 폴더에 정상 배치했다.
- [ ] 해당 폴더에서 `claude` 또는 `claude "/start-1-1"` 실행에 성공했다.
- [ ] 모듈/학습목표 조회 질문 2개 이상으로 컨텍스트 로딩을 검증했다.
- [ ] 다운로드/압축 실패 시 대안 경로(수동 다운로드)를 테스트했다.
- [ ] 온보딩용 부트스트랩 절차를 팀 위키에 복제했다.

## 6) 산출물
