## 1) 목적
- 코스 저장소 준비부터 첫 모듈 실행까지를 한 번에 완료하는 부트스트랩 절차를 정착시킨다.
- 경로/컨텍스트 오류를 줄여 `/start-*` 명령이 정상 동작하는 환경을 확보한다.

## 2) 원문 핵심 요약 (ccforpms)
- 기본 목표는 PM이 복잡한 git 절차를 직접 암기하지 않고, Claude에게 작업 의도를 말해 실행하는 패턴을 익히는 것이다.
- 권장 기본 흐름이 제시된다: `~/Documents` 이동 → 코스 zip 다운로드 → `claude-code-course`로 압축 해제 → `claude "/start-1-1"` 실행.
- 표준 명령 체인이 구체적으로 제공된다: `cd`, `curl -L ... -o course.zip`, `unzip`, `claude` 순서.
- 대안 흐름도 존재한다: 먼저 `claude` 실행 후 다운로드/압축 해제를 요청하고, 폴더 재진입해 학습을 시작한다.
- 장애 대응 포인트가 명확하다: `curl`/`unzip` 미설치, 다운로드 실패, 저장 위치 변경 시 수동 다운로드로 우회 가능.
- 시작 후 검증 질문 예시가 제공된다: "Show me all the Level 1 modules", "List the learning objectives for Module 1.3".
- `/start` 명령이 정상 동작하려면 반드시 `claude-code-course` 폴더 내부 컨텍스트에서 실행해야 한다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선 + Level 2**: 저장소 준비를 이론 설명 대신 즉시 실행 가능한 커맨드 체인으로 제공해 진입 시간을 단축한다.
- **P2 맥락 우선 + Level 2**: 올바른 작업 디렉토리에서 시작하도록 강제해 컨텍스트 누락으로 인한 품질 저하를 막는다.
- **P8 Connected Hub + Level 1**: 시작 단계부터 문서·명령·실행 결과를 연결해 이후 MCP/협업도구 연동으로 자연스럽게 확장한다.
- **P9 Words-to-Workflow + Level 2**: 자연어 요청으로 다운로드/정리/검증까지 이어지는 흐름을 체득하게 만든다.

## 4) 실무 적용 시나리오
신규 입사 PM이 첫날 과제로 코스 환경을 셋업해야 한다. 문서에 제시된 커맨드 체인을 그대로 실행해 `/start-1-1`까지 도달하고, 이어서 레벨/모듈 확인 질문으로 컨텍스트 정상 로딩을 검증한다. 실패 시 수동 다운로드로 우회하고, 어떤 단계에서 막혔는지 체크리스트에 기록해 다음 입사자 온보딩 자료로 축적한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 부트스트랩 표준 명령(`cd → curl → unzip → claude`)을 팀 공통으로 실행하고, 개인별 작업 경로를 동일한 루트로 맞춥니다.
- **수~목 실행**: `/start-1-1` 실행 후 모듈 조회 질문 2~3개로 컨텍스트 로딩을 검증하고 실패 케이스는 원인별로 분류합니다.
- **금요일 검증**: 신규 사용자 관점 리허설(처음부터 다시 셋업)을 진행해 문서만으로 재현 가능한지 점검합니다.
- **Done 기준**: 팀원 전원이 `claude-code-course` 내부에서 시작 명령 실행에 성공하고, 실패/우회 절차가 온보딩 문서에 반영됩니다.
- **리스크/완화**: 네트워크 제한이나 압축 해제 도구 부재로 초기 실패가 발생할 수 있어, 수동 다운로드 링크와 대체 압축 해제 도구 안내를 사전 제공합니다.

## 5) 실습/적용 체크리스트
- [ ] 코스 파일을 `claude-code-course` 폴더에 정상 배치했다.
- [ ] 해당 폴더에서 `claude` 또는 `claude "/start-1-1"` 실행에 성공했다.
- [ ] 모듈/학습목표 조회 질문 2개 이상으로 컨텍스트 로딩을 검증했다.
- [ ] 다운로드/압축 실패 시 대안 경로(수동 다운로드)를 테스트했다.
- [ ] 온보딩용 부트스트랩 절차를 팀 위키에 복제했다.

## 6) 산출물
- 부트스트랩 실행 로그(성공/실패 단계 포함)
- 코스 시작 검증 기록(`/start-1-1` 실행 결과)
- 장애 대응 메모(수동 다운로드 우회 절차 포함)
- 온보딩 재현성 점검표(초기 셋업 소요시간·실패 단계·재현 성공 여부)
