## 1) 목적
- 출시 직전 필요한 마케팅 자산(App Store/소셜 광고/런칭 공지)을 **짧은 시간 안에 패키지로 완성**한다.
- 단일 이미지 제작이 아니라, 채널별 A/B 테스트 가능한 **운영형 자산 세트**를 구축한다.
- 브랜드 일관성과 전환 성능을 동시에 확보하는 제작 기준을 정립한다.

## 2) 원문 핵심 요약 (ccforpms)
1. Nano Banana 마지막 모듈로, 런칭 실무에서 가장 자주 쓰는 그래픽 3종을 다룬다.
2. App Store/Play Store는 규격이 달라서, 처음부터 플랫폼 기준 dimensions로 프롬프트를 작성해야 한다.
3. 스토어 자산은 device frame, 실제 UI, 사용 맥락, 간결한 배경이 핵심이다.
4. Social ad는 피드 주목도 확보가 중요하므로 강한 대비·명확한 가치 메시지·텍스트 여백 설계가 필요하다.
5. 광고 콘셉트는 문제→해결, 제품 중심, 사용자 맥락, 수치 강조, 후기형 등으로 목적에 맞게 선택한다.
6. Launch announcement는 product hero 중심 + 출시 이벤트 무드 + 브랜드 일관성을 동시에 반영해야 한다.
7. 실무에서는 variant를 여러 개 만들고 A/B 테스트 가능하게 구조화해야 성과 개선이 가능하다.
8. `Gemini 3 Pro + Claude Code + style library` 조합으로 속도와 품질을 누적 개선하는 운영이 권장된다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선**: 설명서보다 “오늘 바로 배포 가능한 에셋팩”을 산출물 기준으로 둔다.
- **P2 맥락 우선**: 채널/타깃/CTA/규격 맥락을 프롬프트에 포함해 전환 가능성을 높인다.
- **P4 속도-검증 균형**: 빠른 초안 생성 후 A/B 실험 설계로 검증 루프를 닫는다.
- **P6 자동화 운영 기반**: 런칭 때마다 반복되는 제작을 템플릿화해 운영 부하를 낮춘다.
- **P8 Connected Hub**: 시각 자산 생성 → 채널 게시 → 성과 리뷰를 한 루프로 연결한다.
- **P9 Words-to-Workflow**: 자연어 캠페인 기획을 규격화된 채널 자산으로 즉시 변환한다.
- v2.4의 OMTM 관점에서 자산 평가는 미적 품질만이 아니라 CTR/전환/재방문 지표로 이어져야 한다.

## 4) 실무 적용 시나리오
**상황: 신규 서비스 오픈 D-2 마케팅 스프린트**
1. 핵심 메시지 3개(문제/해결/차별점)를 먼저 확정한다.
2. App Store용 feature graphic과 스크린샷 스타일을 생성한다.
3. 소셜 광고는 3콘셉트 × 2카피로 variant를 만든다.
4. 런칭 공지용 hero 이미지는 보도자료/뉴스레터/링크드인 공통 사용을 전제로 만든다.
5. style ID를 고정해 전체 자산의 톤을 통일한다.
6. 게시 후 48시간 CTR/클릭후전환률을 보고 다음 실험안을 즉시 업데이트한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 출시 메시지 3개(문제, 해결, 차별점)와 채널 규격(App Store, 소셜, 런칭 공지)을 확정하고 에셋 브리프를 작성한다.
- **수~목 실행**: 스토어/광고/공지 자산을 채널별로 생성하고 광고는 3콘셉트 × 2카피 이상으로 variant를 구성해 A/B 테스트안을 만든다.
- **금요일 검증**: 내부 사전 리뷰에서 CTA 명확성, 텍스트 가독성, 브랜드 톤 일관성을 점검해 게시 가능한 최종 패키지를 확정한다.
- **Done 기준**: 채널별 런칭 에셋 패키지와 A/B 테스트 설계안이 준비되고, 게시 후 48시간 성과 수집 계획까지 세팅되어 있다.
- **리스크/완화**: 규격 미스매치 리스크는 채널별 사이즈 체크리스트로 방지하고, 성과 부진 리스크는 첫 48시간 내 카피/비주얼 교체 기준을 사전에 정의해 완화한다.

## 5) 실습/적용 체크리스트
- [ ] 채널별 규격(App Store/소셜/공지)을 표로 정리했다.
- [ ] 각 채널 자산에 핵심 메시지 1개와 CTA를 연결했다.
- [ ] 광고 변형안(최소 3개)과 A/B 테스트 가설을 만들었다.
- [ ] 텍스트 오버레이 여백을 고려한 레이아웃으로 생성했다.
- [ ] style library 재사용으로 브랜드 톤을 통일했다.
- [ ] 게시 후 성과 지표(CTR, 전환, 반응률) 수집 계획을 수립했다.

## 6) 산출물
- 채널별 런칭 에셋 패키지(스토어/광고/공지)
- A/B 테스트 설계안(가설/변수/성공지표)
- 스타일 일관성 체크리스트
- 48시간 성과 리뷰 템플릿
- 채널별 게시 캘린더 및 롤백 플랜(교체 조건, 담당자, 대응 SLA)
