## 1) 목적
- 설치 실패로 학습이 중단되지 않도록 PM 기준의 최소 기술 셋업 절차를 표준화한다.
- 설치 완료 상태를 명령어로 검증해 팀 내 환경 편차를 줄인다.

## 2) 원문 핵심 요약 (ccforpms)
- 필수 전제는 Claude Pro 또는 Max 구독이며 무료 플랜으로는 Claude Code를 사용할 수 없다.
- 런타임 최소 버전이 명확하다: Node.js v18 이상, 지원 OS는 macOS/Windows 10+.
- 설치 순서가 구체적이다: `node --version` 확인 → 필요 시 Node 설치 → 터미널 재시작 → Claude Code 설치.
- macOS 설치 명령은 `curl -fsSL https://claude.ai/install.sh | bash`, Windows PowerShell 설치 명령은 `irm https://claude.ai/install.ps1 | iex`다.
- 설치 검증은 `claude --version`, 계정 인증은 `claude` 실행 후 "Claude account with subscription" 선택으로 진행한다.
- `/doctor` 명령으로 설치 상태, 구독 연결, 시스템 설정, 일반 오류를 일괄 진단한다.
- 대표 장애 패턴이 정리돼 있다: `claude` 미인식, 구독 미연결, Node 버전 불일치, 권한 부족.
- PATH 이슈 시 `~/.bashrc` 또는 `~/.zshrc`에 `export PATH=$HOME/.local/bin:$PATH` 추가가 필요할 수 있다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선 + Level 2**: 설치를 설명형 문서가 아니라 명령 실행과 검증 체크 중심으로 구성해 즉시 가동 상태를 만든다.
- **P4 속도-검증 균형 + Level 2**: 빠른 설치 후 `/doctor`로 검증 게이트를 넣어 초기 오류를 조기 차단한다.
- **P6 자동화 운영 기반 + Level 2**: 환경 진단을 수동 감으로 처리하지 않고 표준 명령 기반 루틴으로 운영한다.
- **P8 Connected Hub + Level 1**: 구독/CLI/OS 권한을 연결된 실행 조건으로 다뤄 이후 MCP·자동화 확장의 토대를 만든다.

## 4) 실무 적용 시나리오
주니어 PM이 주간 회의 전에 PRD 자동화 실습을 해야 하는데 CLI가 동작하지 않는 상황이다. 0.1 문서 절차대로 Node 버전부터 재확인하고, 설치 명령 재실행 후 `/doctor` 결과를 캡처해 팀 채널에 공유한다. 동일 이슈가 반복되면 공통 설치 스크립트와 FAQ를 만들어 다음 온보딩에서 재사용한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 팀원 전원이 `node --version`, `claude --version`을 점검하고 OS별 설치 경로(macOS/Windows)를 표준 문서로 통일합니다.
- **수~목 실행**: 설치 실패 케이스를 의도적으로 1건 재현해 복구 절차(PATH/권한/재설치)를 검증하고 스크린샷과 명령 로그를 수집합니다.
- **금요일 검증**: `/doctor` 결과를 일괄 리뷰해 공통 오류 패턴을 추출하고 다음 온보딩에 반영할 FAQ 우선순위를 확정합니다.
- **Done 기준**: 전원 계정 인증 완료, `/doctor` 치명 오류 0건, 설치/복구 절차 문서가 팀 위키에 게시된 상태입니다.
- **리스크/완화**: 개인 PC 권한·보안 정책 차이로 지연될 수 있으므로, 사전 관리자 권한 체크와 대체 장비(공용 머신) 플랜을 준비합니다.

## 5) 실습/적용 체크리스트
- [ ] `node --version` 결과가 v18 이상임을 확인했다.
- [ ] `claude --version`이 정상 출력된다.
- [ ] `claude` 실행 후 구독 계정 인증까지 완료했다.
- [ ] `/doctor` 결과에서 치명 오류가 0건이다.
- [ ] 설치 로그와 오류 대응 이력을 팀 문서에 남겼다.

## 6) 산출물
- 설치 검증 로그(`node --version`, `claude --version`, `/doctor`)
- OS별 설치/복구 체크리스트
- 자주 발생하는 오류 대응 문서 1부
- 온보딩 설치 상태표(팀원별 완료 여부·블로커·해결일)
