## 1) 목적
- `@` 파일 참조, 이미지 붙여넣기, output style 재사용을 통해 PM의 반복 커뮤니케이션 작업을 빠르게 자동화한다.
- 첫 실습에서 바로 재사용 가능한 결과물 템플릿을 만들어 이후 업무에 전이시킨다.

## 2) 원문 핵심 요약 (ccforpms)
- 모듈 시작 명령은 `/start-1-3`, 권장 학습 시간은 약 30분이며 선행 모듈은 1.1·1.2다.
- 핵심 역량 5가지가 명시된다: 파일 읽기/쓰기, 콘텐츠 분석, 이미지 붙여넣기, 시각 분석, output style.
- `@` 표기는 파일 지정의 기본 규칙이며 단일/복수 파일과 폴더 단위 작업을 모두 처리할 수 있다.
- 자주 다루는 파일 포맷 범위가 넓다: `.md`, `.txt`, `.csv`, `.json`, `.pdf`, 이미지(`.png/.jpg/.gif`).
- 이미지 입력은 OS와 무관하게 터미널에서 `Ctrl+V`를 사용한다(Mac 포함).
- 동일 소스를 Slack/Email/Notion 등 채널별 톤과 깊이로 변환해 이해관계자 커뮤니케이션 효율을 높인다.
- output style을 만들어두면 Executive Briefing, User Story, Weekly Update 같은 문서를 반복 생성할 수 있다.
- 오류 대응 기본은 파일명 대소문자, 경로 정확성, 이미지 크기, 저장 위치 점검이다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선 + Level 1**: 첫 태스크를 파일 생성/변환 중심으로 설계해 즉시 체감 성과를 만든다.
- **P4 속도-검증 균형 + Level 1**: 결과물을 채널별로 변환한 뒤 비교 검토해 빠른 생산과 품질 점검을 동시에 수행한다.
- **P6 자동화 운영 기반 + Level 2**: output style 템플릿을 자산화해 반복 업무를 표준 프로세스로 전환한다.
- **P7 Automation vs Augmentation + Level 1**: 초안 생성은 자동화하되 최종 메시지 톤/우선순위 판단은 PM이 맡도록 분리한다.
- **P9 Words-to-Workflow + Level 2**: 자연어 지시(`@` + 변환 요청)가 곧바로 실무 산출물 생성 흐름으로 이어지게 한다.

## 4) 실무 적용 시나리오
주니어 PM이 주간 업데이트를 Slack, 이메일, 위키에 각각 올려야 하는 상황이다. 원본 회의 노트와 지표 파일을 `@`로 묶어 한 번에 입력하고, 채널별 출력 형식으로 3종 초안을 생성한다. 이후 팀장 검토 포인트(리스크/우선순위)만 수동 보정해 게시하면, 기존 대비 작성 시간을 절반 이하로 줄일 수 있다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 반복 커뮤니케이션 업무 1개(예: 주간 업데이트)를 선정하고 입력 파일 세트(`@회의노트`, `@지표`)를 표준 묶음으로 정의합니다.
- **수~목 실행**: 동일 입력으로 Slack/Email/Notion 버전 3종을 생성해 톤, 길이, 의사결정 정보 포함 여부를 비교합니다.
- **금요일 검증**: 실제 발행본 1건을 채널별로 배포하고 수신자 피드백(이해도·추가질문 건수)을 수집해 템플릿을 개선합니다.
- **Done 기준**: output style 1종 이상이 재사용 가능한 형태로 정리되고, 동일 작업의 작성 시간이 기준 대비 감소합니다.
- **리스크/완화**: 채널별 문맥 차이로 메시지가 부정확해질 수 있으므로, 발행 전 공통 검수 항목(숫자/일정/담당자)을 체크리스트화합니다.

## 5) 실습/적용 체크리스트
- [ ] `/start-1-3` 실행 후 `@` 기반 단일/복수 파일 참조를 모두 테스트했다.
- [ ] 동일 소스를 Slack/Email/Notion 3개 포맷으로 변환했다.
- [ ] 이미지 `Ctrl+V` 입력으로 시각 분석 결과를 1회 이상 생성했다.
- [ ] output style 1개(예: Weekly Update)를 정의하고 재사용했다.
- [ ] 오류 대응 체크(경로/대소문자/저장 위치)를 표준화했다.

## 6) 산출물
- 채널별 업데이트 초안 3종
- output style 템플릿 1종
- 이미지 기반 인사이트 노트 1건
- 첫 태스크 실행/오류 대응 로그
- 채널별 검수 체크리스트(숫자·일정·요청사항 누락 방지)
