## 1) 목적
- 작업 성격에 맞춰 입력 모드와 분석 강도를 선택해, Claude Code 세션 운영 효율을 높인다.
- 속도 옵션과 안전 옵션의 트레이드오프를 이해해 실무 리스크를 통제한다.

## 2) 원문 핵심 요약 (ccforpms)
- 내비게이션 핵심 축은 3가지다: 입력 모드 선택, 분석 깊이 제어, 실행 속도 최적화.
- 입력 모드는 `Edit`, `Auto-Accept`, `Plan`이며 `Shift+Tab`으로 순환 전환한다.
- `Edit`는 diff 확인 후 승인하는 방식으로 중요 파일/학습 단계/안전성 우선 작업에 적합하다.
- `Auto-Accept`는 승인 없이 즉시 반영되어 단순·저위험 작업에서 속도가 빠르다.
- `Plan`은 실행 전 todo를 먼저 수립해 3단계 이상 복합 작업(조사→종합→산출물)에 강하다.
- 분석 강도 키워드가 있다: `think about`, `think harder about`, `ultrathink about`.
- 권한 스킵 실행 옵션 `claude --dangerously-skip-permissions`은 속도를 높이지만 리스크가 크다.
- 반복 사용 시 alias 예시(`cc="claude --dangerously-skip-permissions"`)가 가능하다.
- 실행 중 `Esc`로 즉시 중단 가능하며, 중단 시점까지 생성된 파일은 유지된다.
- 모듈 시작 명령은 `/start-1-7`, 완료 후 다음 단계는 `/start-2-1`이다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P4 속도-검증 균형 + Level 2**: 모드 선택 기준(복잡도·리스크)을 명시해 빠른 실행과 검증 품질을 동시에 관리한다.
- **P1 실행 우선 + Level 2**: Plan/Edit/Auto-Accept를 상황별로 즉시 적용해 작업 정체를 줄인다.
- **P7 Automation vs Augmentation + Level 2**: 자동 승인(Automation)을 쓸 때는 PM 검증 포인트(Augmentation)를 별도 확보한다.
- **P3 PM 오케스트레이션 + Level 2**: PM이 모드·강도·중단 타이밍을 설계해 실행 흐름을 직접 통제한다.

## 4) 실무 적용 시나리오
시니어 PM이 분기 전략 문서와 주간 운영 리포트를 동시에 처리해야 한다. 전략 문서는 `Plan + think harder`로 구조를 먼저 세우고, 반복 포맷 변환은 `Auto-Accept`로 빠르게 처리한다. 마지막 통합 검토는 `Edit` 모드에서 diff를 확인해 리스크를 줄인다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 주요 업무 3종(전략 문서, 주간 보고, 단순 정리)에 대해 기본 모드(Edit/Plan/Auto-Accept)와 think 강도를 사전 매핑합니다.
- **수~목 실행**: 동일 작업을 두 가지 모드로 각각 수행해 소요시간, 수정 횟수, 품질 차이를 비교 기록합니다.
- **금요일 검증**: 권한 스킵 옵션 사용 로그를 리뷰해 허용 범위(저위험 작업)와 금지 범위(핵심 문서/배포 파일)를 확정합니다.
- **Done 기준**: 팀 공통 모드 선택표와 중단/복구 절차가 문서화되고, 다음 주 작업에서 동일 기준으로 운영 가능합니다.
- **리스크/완화**: 속도 우선 사용이 누적되면 품질 저하가 생길 수 있으므로, 고위험 작업은 Edit 모드 강제와 최종 diff 검토를 필수화합니다.
