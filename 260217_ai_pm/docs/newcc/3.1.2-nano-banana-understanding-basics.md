## 1) 목적
- 이미지 생성의 기본 작동 원리(요청→파라미터→생성→저장)를 이해해, 결과 편차를 줄이는 실무 습관을 만든다.
- 해상도/비율/세션 전략을 표준화해 **속도와 품질을 동시에 관리**한다.
- 감각적 요청이 아닌, 반복 가능한 개선 루프로 전환한다.

## 2) 원문 핵심 요약 (ccforpms)
1. 생성 플로우는 `요청 작성 → Claude의 파라미터 변환 → generate() 실행 → Gemini 생성 → outputs 저장` 순서다.
2. 핵심 파라미터는 `prompt`, `reference_images`, `aspect_ratio`, `resolution`이며 초반은 기본값 시작이 효율적이다.
3. 비율은 사용 채널 중심으로 선택해야 한다: 16:9(슬라이드), 1:1(프로필/SNS), 9:16(모바일/숏폼).
4. 해상도는 창의성보다 속도·픽셀에 영향을 주므로 **1K 탐색 → 2K 최종 → 4K 인쇄** 전략이 권장된다.
5. 좋은 작업 방식은 한 번에 많이 바꾸지 않고 **변수 1~2개씩 조정**하는 반복 개선이다.
6. 같은 session은 맥락 기억에 유리하고, 주제가 크게 바뀌면 `new_session()`으로 분리하는 게 좋다.
7. 자주 발생하는 문제는 비율 혼동(16:9 vs 9:16), 모호한 수정 지시, 과도한 4K 사용으로 인한 지연이다.
8. 해결 원칙은 구체적 요청, 단계적 업스케일, 네트워크/부하 점검, 세션 리셋 순서다.

### Claude Code 핵심 기능 연결 (official 반영)
- **입력→출력 워크플로 가시화(Overview)**: 요청, 파라미터, 결과 저장을 한 흐름으로 관리해 기본기 학습 속도를 높인다.
- **명확한 지시문 기본기(Tutorial: Getting Started)**: 채널/비율/해상도/세션을 먼저 고정하면 시행착오가 크게 줄어든다.
- **정밀 반복 개선(Tutorial: Claude Opus 4.6)**: 변수 1~2개만 바꾸는 미세 조정 지시를 통해 결과 해석 가능성을 높인다.
- **브라우징 기반 참고 수집(Tutorial: Claude for Chrome)**: 채널별 레퍼런스 수집으로 비율·구도 선택의 근거를 빠르게 확보한다.
- **협업 리뷰 루프(Tutorial: Claude Cowork)**: PM-디자이너가 라운드별 변경점과 판단 기준을 공유해 재현성을 높인다.
- **통제 가능한 운영 습관(Tutorial: Financial Services)**: 변경 로그와 버전 기록을 남겨 결과 선택의 감사 가능성을 확보한다.
- **세션 전략 실행(Claude Code)**: 같은 주제는 세션 유지, 주제 전환은 분리하는 규칙을 실무 루틴으로 고정한다.

### ccforpms 단계별 실행 예시 (실전 재현)
1) 작업 목적과 채널 먼저 고정
- 입력 아티팩트: `channel-usage-plan.md`(슬라이드/SNS/숏폼), 전달 메시지 초안
- 프롬프트 조각: "이번 이미지를 어디에 쓸지 기준으로 비율(16:9, 1:1, 9:16) 우선순위를 정리해줘."
- 기대 출력: 채널-비율 매핑표
- 검증 체크포인트: 한 작업에 기본 비율 1개, 대체 비율 1개만 설정한다.

2) 기본 파라미터 세팅
- 입력 아티팩트: 기본 프롬프트, 세션 정책 메모
- 프롬프트 조각: "초기 탐색용 파라미터(prompt/aspect_ratio/resolution/session)를 안전 기본값으로 제안해줘."
- 기대 출력: `baseline-params.md`
- 검증 체크포인트: 초기 해상도가 1K로 설정되어 있다.

3) 1차 생성(1K) 3건 실행
- 입력 아티팩트: 기본 파라미터, 프롬프트 3개
- 프롬프트 조각: "동일 주제에서 구도만 다르게 3개 시안을 생성해줘."
- 기대 출력: `outputs/basics-round1-*.png`
- 검증 체크포인트: 변수는 구도 1개만 변경되고 나머지는 고정된다.

4) 변수 1~2개만 변경한 2차 반복
- 입력 아티팩트: 1차 결과, 수정 요청 메모
- 프롬프트 조각: "선택안에서 색감 대비만 높이고 텍스처는 유지해줘."
- 기대 출력: `outputs/basics-round2-*.png`
- 검증 체크포인트: 변경 변수 로그가 남아 있고 비교 가능하다.

5) 세션 유지/분리 결정
- 입력 아티팩트: 주제 변경 여부, 결과 일관성 점수
- 프롬프트 조각: "현재 작업은 세션 유지가 유리한지, new_session 분리가 유리한지 근거와 함께 판단해줘."
- 기대 출력: `session-decision-note.md`
- 검증 체크포인트: 주제 전환 시 세션 분리 이유가 기록된다.

6) 최종안 업스케일(2K)
- 입력 아티팩트: 선택된 1K 안
- 프롬프트 조각: "선택된 시안을 동일 구도로 2K로 재생성해 최종 제출본을 만들어줘."
- 기대 출력: `outputs/basics-final-2k.png`
- 검증 체크포인트: 1K와 2K 결과의 메시지 일관성이 유지된다.

7) 재사용 가능한 기본 가이드 축적
- 입력 아티팩트: 전체 실험 로그, 파라미터 기록
- 프롬프트 조각: "이번 실험에서 유효했던 파라미터 조합과 피해야 할 조합을 표로 정리해줘."
- 기대 출력: `basics-param-playbook.md`
- 검증 체크포인트: 다음 작업에서 즉시 복붙 가능한 템플릿 3개 이상 확보.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P2 맥락 우선**: 품질은 프롬프트 길이가 아니라 비율/해상도/세션 컨텍스트 정합성에서 올라간다.
- **P4 속도-검증 균형**: 1K로 빠르게 가설 검증 후, 최종본만 2K 이상으로 확정한다.
- **P6 자동화 운영 기반**: 파라미터 선택 규칙을 문서화해 팀 내 재현성을 확보한다.
- **P7 Automation vs Augmentation**: 생성과 저장은 자동화, 어떤 변수부터 바꿀지 우선순위는 PM이 결정한다.
- **P9 Words-to-Workflow**: 자연어 요청을 파라미터 관점으로 해석해 실행 가능한 작업 명세로 변환한다.
- 6주 커리큘럼 관점에서 이 단계는 Week 1~2의 “기본 실행력”을 시각 제작 워크플로로 확장하는 브리지다.

## 4) 실무 적용 시나리오
**상황: 서비스 소개 페이지용 히어로 이미지 제작**
1. 먼저 1K/16:9로 초안을 3개 뽑아 메시지 전달력을 비교한다.
2. 선택안에서 색감만 조정하고, 다음 반복에서 구도만 조정한다.
3. 주제 전환(예: 제품컷→라이프스타일컷)이 필요하면 세션을 분리한다.
4. 최종안은 2K로 재생성해 랜딩과 발표 자료에 동시에 사용한다.
5. 모든 시도에 파라미터와 결과 코멘트를 남겨 이후 유사 작업 재사용률을 높인다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 채널별 기본 비율(16:9, 1:1, 9:16)과 해상도 정책(1K 탐색, 2K 최종)을 팀 표준으로 정의하고 샘플 6건을 생성한다.
- **수~목 실행**: 변수 1~2개씩 변경하는 반복 실험을 최소 3라운드 수행해 어떤 변수(색감/구도/조명)가 결과에 가장 크게 영향을 주는지 기록한다.
- **금요일 검증**: 동일 과제를 다른 팀원이 재현해 동일한 품질 수준이 나오는지 확인하고, 재현 실패 원인을 파라미터 가이드에 반영한다.
- **Done 기준**: 비율/해상도/세션 운영 규칙이 문서화되어 있고, 1K 탐색본과 2K 최종본을 재현 가능한 로그와 함께 확보했다.
- **리스크/완화**: 4K 과사용으로 인한 지연 리스크는 최종 단계에서만 허용하고, 비율 혼선은 작업 시작 전 채널-비율 매핑 체크를 의무화해 완화한다.

### 자주 막히는 지점과 해결 가이드
1) 블로커: 16:9와 9:16이 자주 뒤바뀜
- 근본 원인: 채널 목적보다 감각적으로 비율 선택
- 해결 방법: 작업 시작 전 채널-비율 체크박스 1회 의무 확인

2) 블로커: 반복 실험해도 무엇이 좋아졌는지 모름
- 근본 원인: 변경 변수 기록이 없음
- 해결 방법: 라운드마다 변경 변수/결과 점수/다음 액션을 로그화

3) 블로커: 세션이 꼬여 스타일이 갑자기 변함
- 근본 원인: 주제 변경에도 동일 세션을 계속 사용
- 해결 방법: 주제 전환 기준(메시지/대상/채널 변화) 충족 시 즉시 new_session

4) 블로커: 고해상도 집착으로 작업 속도 급감
- 근본 원인: 탐색 단계부터 2K/4K 사용
- 해결 방법: 탐색 1K 고정, 최종 확정 후 업스케일 원칙 고정

- [ ] 1K 초안 탐색 후 2K 최종 확정 흐름을 실제로 적용했다.
- [ ] 변수 1~2개 단위 반복 개선을 최소 3회 수행했다.
- [ ] 세션 유지/세션 분리 기준을 작업 로그에 기록했다.
- [ ] 비율 오류/지연 문제를 스스로 진단하고 수정했다.
- [ ] 결과 선택 이유(전달력, 일관성, 사용처 적합성)를 문장으로 남겼다.

## 6) 산출물
- 변수별 생성 실험표(비율/해상도/세션)
- 1K 탐색본 + 2K 최종본 세트
- 반복 개선 로그(변경 항목/결과/판단 근거)
- 재사용 가능한 파라미터 선택 가이드
- 채널별 권장 세팅 매트릭스(목적, 비율, 해상도, 세션 전략, 금지 패턴)
