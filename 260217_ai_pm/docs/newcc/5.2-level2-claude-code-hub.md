# 5.2 Level 2 - Claude Code Hub

**빠른 요약**
- 5.2 Level 2 - Claude Code Hub 주제를 빠르게 이해하고 바로 써볼 수 있게 정리한 문서입니다.

**이번 문서 목표**
- 읽고 나면 5.2 Level 2 - Claude Code Hub 내용을 내 업무 흐름에 맞게 적용할 수 있습니다.

**처음 보는 분 가이드**
- 먼저 `## 1) 목적`과 `## 5) 실습/적용 체크리스트`만 읽고 시작하세요.
- 용어가 낯설면 [사전](./사전.md)을 먼저 확인하세요.

## 1) 목적
- Claude Code를 팀의 실행 허브로 표준화해, 문서-코드-운영 작업을 같은 프로토콜로 처리한다.
- Level1 산출물을 재사용 가능한 운영 자산(CLAUDE.md, Slash Commands, 템플릿)으로 승격한다.
- 사람 의존적인 반복 대응을 줄이고, 팀 단위 재현성을 확보한다.

## 2) 원문 핵심 요약 (ccforpms)
- 원문 1.7은 작업 성격에 따라 Edit/Auto-Accept/Plan 모드를 선택하는 운영법을 제시한다.
- 원문 1.4는 병렬 agent를 활용해 대량 리서치/문서 생성을 fan-out 후 synthesis하는 패턴을 권장한다.
- 원문 1.5는 반복 역할을 custom sub-agent로 고정해 리뷰 품질과 속도를 높인다.
- 원문 1.6/Project memory 맥락은 CLAUDE.md 기반 지속 컨텍스트 유지의 중요성을 강조한다.
- 원문 2.x는 PRD/분석/전략을 파일 산출물로 남기는 실행형 패턴을 반복한다.

## 3) 이든 철학 반영 포인트 (v2.4)
### Dennis baseline vs 이든 확장
| 항목 | Dennis baseline | 이든 Level2 운영 |
|---|---|---|
| 실행 허브 | Cursor 중심 | Claude Code + CLAUDE.md + Slash Commands |
| 병렬 처리 | ad-hoc 중심 | 병렬 + 역할 고정(sub-agent) + 품질 게이트 |
| 운영 자산 | 개인 노하우 | 팀 공용 템플릿/명령어/컨텍스트 정책 |
| 추적성 | 대화 로그 중심 | 파일 기반 산출물 + 링크 기반 추적 |

- **FR-017**(PM 허브 인프라) 직접 반영: CLAUDE.md, 폴더 구조, 필수 명령 셋 정착.
- **P2/P9**: 맥락 문서를 코드처럼 운영하고, 문장을 실행 가능한 명령으로 전환.
- **NFR-009**: Claude.ai/Claude Code/n8n/MCP 간 단절 없는 데이터 흐름 확보.

## 4) 실무 적용 시나리오
### 실행 순서 (1주)
1. **허브 초기화**
   - 프로젝트 CLAUDE.md에 작업 원칙, 금지사항, 산출물 규격 정의
   - 폴더: `tasks/`, `research/`, `reports/`, `templates/`
2. **명령어 표준화**
   - `/today`, `/prd`, `/status`, `/agent` 최소 4개 운영
   - 각 명령어 입력/출력 포맷 고정
3. **역할 분리 운영**
   - Researcher / Reviewer / Synthesizer sub-agent 정의
   - 병렬 실행 후 최종 통합 문서 1개로 귀결
4. **품질 게이트 도입**
   - 문서 산출물 필수 항목(목표/근거/지표/다음 액션)
   - 검수 미통과 시 재작성 규칙 적용

### 실패 패턴
- 실패1: CLAUDE.md가 선언문에 그치고 실행에 미반영
  - 대응: 명령어/템플릿이 참조하는 필수 변수로 연결
- 실패2: 병렬 결과가 통합되지 않음
  - 대응: synthesis 담당 role 고정, 단일 최종본 원칙
- 실패3: 모드 선택 오류로 품질 저하
  - 대응: 3단계 이상 복합 작업은 Plan 모드 강제
- 실패4: 요청/응답 포맷 불일치
  - 대응: 요청 템플릿과 보고 템플릿 버전 고정

### 이번 주 완료형 실행 플랜
- **월~화:** 프로젝트 CLAUDE.md와 폴더 표준(`tasks/research/reports/templates`)을 확정하고 기본 커맨드 4개를 연결한다.
- **수~목:** Researcher/Reviewer/Synthesizer 역할을 실제 문서 작업 1건에 적용해 병렬-통합 흐름을 검증한다.
- **금 검증:** 처리시간·재작업률·검수 누락률을 측정해 Hub 운영의 기준선(baseline)을 산출한다.
- **Done 기준:** CLAUDE.md 반영본, 커맨드 테스트 로그, sub-agent 정의서, 품질 게이트 적용 결과가 모두 남는다.
- **리스크·완화:** 명령 포맷 불일치로 실패할 수 있으므로 입력/출력 예시를 템플릿에 고정하고 버전 태그를 부여한다.

## 5) 실습/적용 체크리스트
- [ ] 프로젝트 CLAUDE.md에 운영 원칙과 금지 규칙을 명시했다.
- [ ] `/today`, `/prd`, `/status`, `/agent` 명령을 테스트 완료했다.
- [ ] sub-agent 2개 이상을 역할별로 정의했다.
- [ ] 산출물 품질 게이트(필수 섹션/지표/근거)를 운영에 적용했다.
