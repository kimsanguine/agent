## 1) 목적
- Claude Cowork를 “채팅 도구”가 아닌 **파일 기반 실행 파트너**로 운영하는 기준을 제공한다.
- 비개발 PM이 대량 문서/리서치/정리 업무를 위임할 때 실패율을 낮춘다.
- 추후 `5.2`/`5.6` 통합 시, 남길 내용과 삭제할 내용을 명확히 분리한다.

## 2) 원문 핵심 요약 (ccforpms)
- Cowork는 Claude Code의 비개발자 친화 GUI 형태로, 폴더 내 파일을 직접 읽고 생성/편집할 수 있다.
- 일반 채팅과 다르게 `.xlsx`, `.pptx`, `.docx` 등 실제 업무 산출물을 바로 만들 수 있는 점이 핵심 가치다.
- 시작 조건은 macOS + Claude Desktop + Claude Pro이며, 초기에는 작업 전용 폴더 1개부터 연결하는 운영이 권장된다.
- 성과를 좌우하는 프레임은 Done Framework(완료 정의, 필수 맥락, 제약 조건)로 제시된다.
- 활용 영역은 파일 정리, 리서치 통합, 문서 생성, 데이터 추출, 배치 처리다.
- Sub-agents, Skills, Chrome 연계로 확장 가능하지만, 세션 메모리 부재·일부 커넥터 불안정 등 제약도 존재한다.
- 안전 수칙은 백업, 민감정보 분리, 삭제 금지 제약 명시가 기본이다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선**: Cowork 사용 결과는 반드시 파일 산출물로 남기고, 링크와 버전까지 기록한다.
- **P2 Context-rich**: 요청 시 입력 데이터 범위/완료 조건/금지사항을 먼저 고정해 재작업률을 줄인다.
- **P7 Automation vs Augmentation**: 문서 포맷 변환·정리·초안 생성은 자동화, 최종 의사결정/대외 발신은 사람 검수로 분리한다.
- **P8 Connected Hub 정합성 검사**: Cowork 단독 운영은 임시로 허용하되, 장기적으로 `5.2(Claude Code Hub)`에 흡수 가능한지 평가한다.
- **NFR-010/011 관점**: 자동화 안정성과 관측성(실패 기록, 재실행 로그)이 없는 경우 확장 적용을 제한한다.

## 4) 실무 적용 시나리오
### 시나리오: 주간 리서치 정리 배치 작업
1. 입력 폴더를 `inbox/processed/outputs/reference`로 분리한다.
2. 시작 프롬프트에 완료 정의를 명확히 작성한다.
   - 예: “30개 문서를 카테고리 5개로 분류하고, 근거 문장 1개씩 포함한 요약표를 작성. 원본 삭제 금지.”
3. Cowork로 1차 분류/요약을 실행한다.
4. PM이 표본 20%를 검수해 오분류율을 기록한다.
5. 오분류율이 기준(예: 10%) 이하면 팀 공유본으로 배포, 초과하면 규칙 보강 후 재실행한다.

### 이번 주 완료형 실행 플랜
- **월~화:** `inbox/processed/outputs/reference` 폴더 구조를 팀 공용으로 만들고 Done Framework 프롬프트 템플릿 1종을 확정한다.
- **수~목:** 실제 리서치 30건을 대상으로 Cowork 분류/요약 배치를 2회 실행하고 오분류 유형을 태깅한다.
- **금 검증:** 표본 20% 검수 결과로 오분류율·누락률을 계산해 기준치(오분류율 10% 이하) 충족 여부를 판단한다.
- **Done 기준:** 템플릿 1종, 배치 로그 2회분, 검수 리포트 1종, 팀 공유본 링크가 모두 연결된다.
- **리스크·완화:** 커넥터 오류나 폴더 권한 이슈가 나면 수동 업로드 fallback 경로를 즉시 사용하고 재실행 시간을 로그에 남긴다.

## 5) 실습/적용 체크리스트
- [ ] Done Framework(완료 정의/맥락/제약) 3요소를 요청 프롬프트에 넣었다.
- [ ] 원본 보존 규칙(삭제 금지/백업 위치)을 명시했다.
- [ ] 결과물 품질 기준(정확성/형식/재현성)과 샘플 검수 비율을 정했다.
- [ ] 실패 시 재시도 횟수와 에스컬레이션 경로를 정의했다.
- [ ] Cowork 결과물을 `5.2` 허브 문서 체계로 넘길 수 있게 링크를 연결했다.

## 6) 산출물
- Cowork 작업 지시 템플릿(Done Framework 내장)
- 배치 처리 결과 로그(입력량/완료시간/오류/재시도)
- 샘플 검수 리포트(오분류율/재작업 사유)
- Cowork→5.2 허브 연계 체크시트(링크 누락/포맷 불일치 점검)

## 통합/삭제 판단 기준
- **통합 권고 조건**
  - 최근 8주 기준 Cowork 단독 문서 참조율 < 20%
  - 동일 내용이 `5.2-level2-claude-code-hub.md`와 70% 이상 중복
  - Cowork 고유 실습 없이도 `5.6` 플레이북으로 업무 수행 가능
- **독립 유지 조건**
  - GUI 기반 사용자(비개발 PM) 비중이 40% 이상
  - Cowork 전용 운영 리스크(권한/커넥터/파일 정책)가 별도 관리 필요
- **삭제/흡수 의사결정 방식**
  - 월간 리뷰 2회 연속 “통합 권고 조건 충족” 시 본 문서를 `5.2` 부록으로 흡수하고 독립 문서 삭제
