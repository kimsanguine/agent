## 1) 목적
- PM 피치/제안서에서 사용자 문제와 제품 가치를 **한눈에 전달하는 5종 비주얼 세트**를 만든다.
- 기능 나열식 설명을 벗어나, 사용자 맥락 중심 스토리텔링으로 설득력을 높인다.
- 제품/마케팅/세일즈 자료에 재사용 가능한 시각 자산 베이스를 구축한다.

## 2) 원문 핵심 요약 (ccforpms)
1. 이 모듈은 이미지 생성 자체보다 **PM 산출물 연결성**이 핵심이다.
2. 실전 구성은 5가지 자산 중심이다: persona portrait, journey map, wireframe transformation, device mockup, lifestyle shot.
3. persona portrait는 공감 형성용이므로 인물 특성/표정/복장/환경/톤을 구체화해야 한다.
4. journey map은 발표용으로 5~7단계가 적절하고, 단계별 pain point를 시각적으로 명확히 드러내야 한다.
5. wireframe transformation은 손스케치를 reference로 넣어 **레이아웃 의도 유지 + 시각 품질 개선**을 동시에 달성한다.
6. device mockup은 기기 프레임/각도/그림자/맥락으로 현실감을 높여 “실제로 존재하는 제품” 인상을 만든다.
7. lifestyle shot은 제품 사용 장면을 스토리로 전달하는 hero asset이며, 텍스트 오버레이 공간을 남겨야 재활용성이 높다.
8. 권장 제작 방식은 1K 빠른 반복 후 최종본 2K 확정이며, style library 재사용으로 덱 전체 톤을 통일한다.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선**: 이론보다 5종 자산을 실제 슬라이드에 배치 가능한 상태로 완성한다.
- **P2 맥락 우선**: 사용자 장면(언제/어디서/왜)과 pain point를 먼저 확정한 뒤 생성한다.
- **P3 PM 오케스트레이션**: PM은 생성자가 아니라 “스토리 순서(사용자→문제→해결→비전)”를 설계·검증한다.
- **P4 속도-검증 균형**: 빠른 시안 생성 후 10초 이해 테스트로 전달력을 검증한다.
- **P5 한국 실무 적합성**: 제안서/내부 보고/세일즈 덱처럼 짧은 시간 설득이 필요한 국내 업무 맥락에 맞춘다.
- **P7 Automation vs Augmentation**: 자산 생성은 자동화, 메시지 우선순위·장표 구성은 PM의 증강 판단으로 수행한다.

## 4) 실무 적용 시나리오
**상황: B2B 솔루션 소개 미팅 24시간 전**
1. 핵심 타깃 1명(persona)과 대표 사용 장면 1개를 확정한다.
2. journey map 6단계로 문제 흐름을 시각화한다.
3. 기존 스케치를 wireframe transformation으로 정돈해 구현 방향을 명확히 한다.
4. 핵심 화면을 device mockup에 얹어 데모 전 기대치를 높인다.
5. lifestyle shot을 첫 슬라이드 hero로 배치해 감정적 진입점을 만든다.
6. 전체 장표를 동일 스타일로 정리해 신뢰감을 확보한다.

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 핵심 사용자 1명과 대표 사용 시나리오 1개를 고정한 뒤 persona/journey/wireframe/mockup/lifestyle 5종 초안을 모두 생성한다.
- **수~목 실행**: 각 자산의 핵심 메시지 1문장을 기준으로 전달력 리뷰를 진행하고, 스토리 흐름(사용자→문제→해결→비전)에 맞춰 장표 배치를 정리한다.
- **금요일 검증**: 내부 10초 이해 테스트를 실행해 메시지 오해가 발생한 장표를 재생성하고 최종 2K 세트를 확정한다.
- **Done 기준**: 5종 자산이 실제 발표 슬라이드에 배치된 상태로 준비되고, 자산별 사용처와 메시지 메모가 함께 정리되어 있다.
- **리스크/완화**: 비주얼은 좋지만 메시지가 약한 리스크는 장표별 핵심 문장 선작성으로 완화하고, 자산 간 톤 불일치는 style ID 고정으로 통제한다.

## 5) 실습/적용 체크리스트
- [ ] 5종 자산(persona/journey/wireframe/mockup/lifestyle) 초안을 모두 만들었다.
- [ ] 각 자산에 전달해야 할 핵심 메시지 1개를 명시했다.
- [ ] journey map 단계 수를 5~7로 제한하고 pain point를 표시했다.
- [ ] wireframe은 low-fi 의도를 유지한 채 시각 품질만 개선했다.
- [ ] 최종 자산은 2K 기준으로 통일 저장했다.
- [ ] 발표 흐름(사용자→문제→해결→비전)에 맞게 실제 슬라이드 배치를 완료했다.

## 6) 산출물
- 5종 PM 시각 자산 세트
- 발표/제안서용 슬라이드 배치안
- 자산별 메시지/사용처 메모
- style 재사용 로그(톤 일관성 검증 포함)
- 10초 전달력 테스트 결과표(자산별 이해도, 오해 포인트, 수정 액션)
