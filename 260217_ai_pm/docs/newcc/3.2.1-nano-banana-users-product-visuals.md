## 1) 목적
- PM 피치/제안서에서 사용자 문제와 제품 가치를 **한눈에 전달하는 5종 비주얼 세트**를 만든다.
- 기능 나열식 설명을 벗어나, 사용자 맥락 중심 스토리텔링으로 설득력을 높인다.
- 제품/마케팅/세일즈 자료에 재사용 가능한 시각 자산 베이스를 구축한다.

## 2) 원문 핵심 요약 (ccforpms)
1. 이 모듈은 이미지 생성 자체보다 **PM 산출물 연결성**이 핵심이다.
2. 실전 구성은 5가지 자산 중심이다: persona portrait, journey map, wireframe transformation, device mockup, lifestyle shot.
3. persona portrait는 공감 형성용이므로 인물 특성/표정/복장/환경/톤을 구체화해야 한다.
4. journey map은 발표용으로 5~7단계가 적절하고, 단계별 pain point를 시각적으로 명확히 드러내야 한다.
5. wireframe transformation은 손스케치를 reference로 넣어 **레이아웃 의도 유지 + 시각 품질 개선**을 동시에 달성한다.
6. device mockup은 기기 프레임/각도/그림자/맥락으로 현실감을 높여 “실제로 존재하는 제품” 인상을 만든다.
7. lifestyle shot은 제품 사용 장면을 스토리로 전달하는 hero asset이며, 텍스트 오버레이 공간을 남겨야 재활용성이 높다.
8. 권장 제작 방식은 1K 빠른 반복 후 최종본 2K 확정이며, style library 재사용으로 덱 전체 톤을 통일한다.

### Claude Code 핵심 기능 연결 (official 반영)
- **문제→해결 스토리 전환(Overview)**: 사용자 맥락을 시각 자산으로 빠르게 변환해 PM 메시지 전달력을 강화한다.
- **초기 지시문 구조화(Tutorial: Getting Started)**: persona, journey, wireframe, mockup, lifestyle 목적을 분리해 산출물 품질을 안정화한다.
- **복합 요구 정렬(Tutorial: Claude Opus 4.6)**: 공감(사용자), 설득(비즈니스), 구현(제품) 요소를 동시에 반영하는 프롬프트를 설계한다.
- **참고자료 수집(Tutorial: Claude for Chrome)**: 유사 사례/레퍼런스를 빠르게 확보해 장표 메시지와 시각 톤의 일치도를 높인다.
- **협업 기반 리뷰(Tutorial: Claude Cowork)**: PM·디자이너·영업이 같은 자산을 보고 전달력 기준을 빠르게 맞춘다.
- **근거 중심 의사결정(Tutorial: Financial Services)**: 자산별 선택 이유와 변경 이력을 남겨 리뷰/승인 논의를 단축한다.
- **실행 중심 반복(Claude Code)**: 1K 탐색→2K 확정 흐름으로 24시간 내 발표 가능 자산 세트를 만든다.

### ccforpms 단계별 실행 예시 (실전 재현)
1) 사용자 스토리와 메시지 고정
- 입력 아티팩트: `persona-notes.md`, `customer-call-summary.md`, `value-proposition.md`
- 프롬프트 조각: "이번 덱에서 전달할 사용자 문제-해결-가치 메시지를 3문장으로 고정해줘."
- 기대 출력: `visual-story-brief.md`
- 검증 체크포인트: 각 문장이 persona/journey/solution 장표와 1:1 대응된다.

2) persona portrait 생성
- 입력 아티팩트: 타깃 페르소나 특성표
- 프롬프트 조각: "직무/환경/감정 상태가 드러나는 persona portrait를 생성하되 과장된 스톡 느낌은 피해줘."
- 기대 출력: `outputs/persona-*.png`
- 검증 체크포인트: 인물 설정이 실제 타깃 사용자와 불일치하지 않는다.

3) journey map 시각화
- 입력 아티팩트: 사용자 여정 단계 메모(5~7단계)
- 프롬프트 조각: "5~7단계 journey map으로 pain point를 시각적으로 강조해줘."
- 기대 출력: `outputs/journey-map-*.png`
- 검증 체크포인트: 단계별 pain point가 아이콘/텍스트로 식별 가능하다.

4) wireframe transformation
- 입력 아티팩트: low-fi 스케치, 핵심 인터랙션 설명
- 프롬프트 조각: "레이아웃 의도를 유지하며 발표용으로 정돈된 high-fidelity 스타일로 변환해줘."
- 기대 출력: `outputs/wireframe-transform-*.png`
- 검증 체크포인트: 핵심 UI 구조가 원본 스케치 대비 훼손되지 않는다.

5) device mockup 제작
- 입력 아티팩트: 핵심 화면 캡처, 브랜드 색상
- 프롬프트 조각: "실제 기기 프레임과 자연스러운 그림자를 적용한 mockup을 만들어줘."
- 기대 출력: `outputs/device-mockup-*.png`
- 검증 체크포인트: 기기 프레임과 화면 비율이 왜곡되지 않는다.

6) lifestyle hero shot 생성
- 입력 아티팩트: 사용 맥락 시나리오, 카피 초안
- 프롬프트 조각: "제품 사용 맥락이 드러나는 lifestyle shot을 만들고 상단 텍스트 여백을 확보해줘."
- 기대 출력: `outputs/lifestyle-hero-*.png`
- 검증 체크포인트: 카피를 얹었을 때 가독성이 유지된다.

7) 슬라이드 통합 및 전달력 검증
- 입력 아티팩트: 5종 자산 전체
- 프롬프트 조각: "사용자→문제→해결→비전 흐름으로 장표 순서를 추천하고 10초 이해 테스트 질문을 만들어줘."
- 기대 출력: `deck-visual-sequence.md`, 테스트 결과
- 검증 체크포인트: 10초 테스트에서 핵심 메시지 오해율이 기준 이하.

## 3) 이든 철학 반영 포인트 (v2.4)
- **P1 실행 우선**: 이론보다 5종 자산을 실제 슬라이드에 배치 가능한 상태로 완성한다.
- **P2 맥락 우선**: 사용자 장면(언제/어디서/왜)과 pain point를 먼저 확정한 뒤 생성한다.
- **P3 PM 오케스트레이션**: PM은 생성자가 아니라 “스토리 순서(사용자→문제→해결→비전)”를 설계·검증한다.
- **P4 속도-검증 균형**: 빠른 시안 생성 후 10초 이해 테스트로 전달력을 검증한다.
- **P5 한국 실무 적합성**: 제안서/내부 보고/세일즈 덱처럼 짧은 시간 설득이 필요한 국내 업무 맥락에 맞춘다.
- **P7 Automation vs Augmentation**: 자산 생성은 자동화, 메시지 우선순위·장표 구성은 PM의 증강 판단으로 수행한다.

## 4) 실무 적용 시나리오
**상황: B2B 솔루션 소개 미팅 24시간 전**
1. 핵심 타깃 1명(persona)과 대표 사용 장면 1개를 확정한다.
2. journey map 6단계로 문제 흐름을 시각화한다.
3. 기존 스케치를 wireframe transformation으로 정돈해 구현 방향을 명확히 한다.
4. 핵심 화면을 device mockup에 얹어 데모 전 기대치를 높인다.
5. lifestyle shot을 첫 슬라이드 hero로 배치해 감정적 진입점을 만든다.
6. 전체 장표를 동일 스타일로 정리해 신뢰감을 확보한다.

### PM Use Case 확장 (Claude Code 기능 매핑)
| PM 상황 | Claude Code 기능 | 실행 예시 | 검증 포인트 |
|---|---|---|---|
| 발표용 사용자 스토리가 추상적임 | 스토리 브리프 구조화 | `"사용자 문제-해결-가치 메시지를 3문장으로 고정"` | 자산 5종과 메시지 1:1 매핑 |
| 장표별 톤이 달라 설득력이 약함 | 스타일 일관화 운영 | `"style ID를 고정해 persona~lifestyle까지 동일 톤으로 생성"` | 장표 간 톤 편차 감소 |
| wireframe이 전달이 안 됨 | reference 기반 고도화 | `"low-fi 스케치 의도 유지 + 발표용 high-fi 변환"` | 핵심 UI 구조 보존 확인 |
| 미팅 직전 자산 검토가 어렵다 | 10초 이해도 테스트 지원 | `"자산별 10초 테스트 질문 생성 및 오해 포인트 정리"` | 오해율 기준 이하 달성 |
| 세일즈/CS와 해석이 다름 | 협업 코워킹 리뷰 | `"영업 관점 질문 3개, CS 관점 질문 3개로 리뷰"` | 메시지 해석 편차 감소 |

### 이번 주 완료형 실행 플랜
- **월~화 실행**: 핵심 사용자 1명과 대표 사용 시나리오 1개를 고정한 뒤 persona/journey/wireframe/mockup/lifestyle 5종 초안을 모두 생성한다.
- **수~목 실행**: 각 자산의 핵심 메시지 1문장을 기준으로 전달력 리뷰를 진행하고, 스토리 흐름(사용자→문제→해결→비전)에 맞춰 장표 배치를 정리한다.
- **금요일 검증**: 내부 10초 이해 테스트를 실행해 메시지 오해가 발생한 장표를 재생성하고 최종 2K 세트를 확정한다.
- **Done 기준**: 5종 자산이 실제 발표 슬라이드에 배치된 상태로 준비되고, 자산별 사용처와 메시지 메모가 함께 정리되어 있다.
- **리스크/완화**: 비주얼은 좋지만 메시지가 약한 리스크는 장표별 핵심 문장 선작성으로 완화하고, 자산 간 톤 불일치는 style ID 고정으로 통제한다.

### 자주 막히는 지점과 해결 가이드
1) 블로커: 페르소나는 좋은데 제품 가치 연결이 약함
- 근본 원인: 사용자 문제와 제품 해결 문장이 분리됨
- 해결 방법: 장표마다 "문제 1줄-해결 1줄" 쌍을 강제 입력

2) 블로커: journey map이 복잡해 한눈에 안 들어옴
- 근본 원인: 단계 수 과다(8단계 이상)
- 해결 방법: 5~7단계로 축소하고 핵심 pain point만 강조

3) 블로커: wireframe 변환 후 원래 의도가 사라짐
- 근본 원인: 시각 개선 과정에서 정보 구조 변경
- 해결 방법: 변환 전 핵심 UI 블록 체크리스트로 구조 보존 검증

4) 블로커: lifestyle shot에 카피를 얹으면 가독성 저하
- 근본 원인: 텍스트 여백과 대비 설계 미흡
- 해결 방법: 생성 시 여백 영역(상단/좌측 %)을 명시하고 대비 강화 지시 추가

## 5) 실습/적용 체크리스트
- [ ] 5종 자산(persona/journey/wireframe/mockup/lifestyle) 초안을 모두 만들었다.
- [ ] 각 자산에 전달해야 할 핵심 메시지 1개를 명시했다.
- [ ] journey map 단계 수를 5~7로 제한하고 pain point를 표시했다.
- [ ] wireframe은 low-fi 의도를 유지한 채 시각 품질만 개선했다.
