# 4.2: 계획
원문: https://ccforpms.com/vibe-coding/plan

## 한 줄 요약
구현 전에 Claude Code와 요구사항·로직·디자인 결정을 정리해 REQUIREMENTS.md로 확정하면, 다음 빌드 단계의 정확도가 크게 올라간다.

## 핵심 내용
- 이 레슨(약 20~25분)은 개발 전에 명확한 spec을 만드는 준비 단계다.
- 요구사항은 협업 기준 문서이며, 구체적일수록 결과가 의도에 맞게 나온다.
- 소프트웨어 로직은 if-then 규칙의 집합이고, PM은 규칙을 자연어로 정의하면 Claude가 코드로 구현한다.
- 추천/퀴즈형 기능은 보통 입력 → 점수화 → 최고 점수 결과 출력의 구조를 따른다.
- 디자인 방향은 말보다 시각 확인이 빠르므로 HTML preview files를 비교해 선택·조합하는 방식이 유효하다.
- 최종 결정(기능, 로직, 스타일, 콘텐츠)은 REQUIREMENTS.md에 저장되며 다음 레슨의 구현 기준이 된다.

## 실습/실행 단계
1. Claude Code에서 `/start-4-2`를 실행해 인터랙티브 인터뷰를 시작한다.
2. 만들 기능의 목적, 요구사항, 콘텐츠 요소를 질문에 맞춰 구체적으로 정리한다.
3. 입력값별 점수 규칙과 결과 산출 방식 등 핵심 로직을 정의한다.
4. HTML preview files로 스타일을 검토한 뒤, 합의한 내용을 REQUIREMENTS.md로 확정한다.

## 기억할 포인트
- 모호한 지시보다 구체적인 spec이 재작업을 줄인다.
- PM의 핵심은 구현보다 의사결정(규칙·매핑·우선순위)이다.
- REQUIREMENTS.md의 품질이 4.3 Build & Iterate 결과 품질을 좌우한다.
