# 3.2.1: 사용자 및 제품 비주얼
원문: https://ccforpms.com/nano-banana/users-product-visuals

## 한 줄 요약
PM 피치에 필요한 5가지 시각 자산(persona portrait, journey map, wireframe, device mockup, lifestyle shot)을 빠르게 만드는 실전 가이드다.

## 핵심 내용
- 이 모듈은 이미지 생성 결과물을 PM 산출물로 연결하는 데 초점이 있다.
- 시나리오는 제조 현장용 앱 TaskFlow Mobile 피치이며, 사용자·문제·해결·비전을 5개 비주얼로 구성한다.
- persona portrait는 타깃 사용자 공감 형성용으로, 인물 특성·복장·환경·톤을 명확히 지정하는 것이 핵심이다.
- journey map은 단계별 흐름과 pain point를 보여주며, 발표용은 5~7단계로 단순화하는 것을 권장한다.
- wireframe transformation은 손스케치를 reference image로 넣어 레이아웃은 유지하고 시각 품질만 정리한다.
- device mockup은 iPhone/Android/MacBook/iPad 등 맥락에 맞는 프레임과 각도·그림자로 현실감을 높인다.
- lifestyle shot은 제품 사용 장면의 스토리 전달용 hero image이며, 텍스트 오버레이 공간을 남기면 활용도가 높다.
- style library를 재사용하면 덱 전체 톤을 통일하고 variant 비교 속도를 높일 수 있다.
- 권장 작업 방식은 1K로 반복 생성 후, 최종본만 2K로 업스케일하는 흐름이다.

## 실습/실행 단계
1. Claude Code에서 `/start-3-2-1`로 시작하고 TaskFlow Mobile 맥락을 확인한다.
2. persona portrait를 생성해 핵심 사용자(예: Marcus Chen)의 이미지 기준을 잡는다.
3. 같은 사용자 기준으로 journey map을 만들고 단계별 pain point를 표시한다.
4. 손그림 스케치를 넣어 wireframe transformation을 수행하고 fidelity를 명시한다.
5. 완성 화면을 device mockup에 적용해 실제 사용 인상을 만든다.
6. 동일 persona가 현장에서 앱을 쓰는 lifestyle shot을 생성해 hero image로 선택한다.
7. 5개 자산을 슬라이드 흐름(사용자→문제→해결→비전)에 맞게 배치하고 variant를 비교해 최종본을 고른다.
8. 다음 모듈은 `/start-3-2-2`로 이어간다.

## 기억할 포인트
- 한 발표 자료에서는 스타일 일관성이 신뢰감을 만든다.
- 첫 결과보다 iteration 품질이 중요하다.
- wireframe은 의도적으로 low-fi를 지정해야 과도한 완성도를 피할 수 있다.
- 결과가 어색하면 나이·표정·복장·각도·조명 조건을 더 구체화해 재생성한다.
- 이 모듈 목표는 완벽함보다 빠른 제작과 명확한 스토리 전달이다.
