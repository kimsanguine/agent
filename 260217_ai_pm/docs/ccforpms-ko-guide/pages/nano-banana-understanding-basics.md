# 3.1.2: 기본 개념 이해하기
원문: https://ccforpms.com/nano-banana/understanding-basics

## 한 줄 요약
Claude가 자연어 요청을 `generate()` 호출로 바꿔 Gemini에 전달하므로, 1K로 빠르게 반복하고 마지막에 2K로 확정하는 방식이 가장 효율적이다.

## 핵심 내용
- 이미지 생성 과정은 요청 작성 → Claude의 파라미터 변환 → `generate()` 실행 → Gemini 생성 → `outputs/` 저장 순서로 진행된다.
- 핵심 파라미터는 `prompt`, `reference_images`, `aspect_ratio`, `resolution`이며, 초반에는 기본값으로 시작해도 충분하다.
- 작업 방식은 기본값에 맡기는 방법과 비율/해상도를 명시하는 방법 두 가지다.
- `aspect_ratio`는 사용처 기준으로 선택한다(예: 16:9 슬라이드, 1:1 프로필, 9:16 모바일 화면).
- 해상도는 창의성보다 픽셀 크기/속도에 영향이 크며, 1K(탐색) → 2K(최종) → 4K(인쇄) 전략이 권장된다.
- 반복 수정(iteration)이 핵심이며, 같은 session에서는 맥락을 기억해 누적 편집이 잘 된다.
- 큰 방향 전환이나 완전히 다른 주제는 `new_session()`으로 새로 시작하는 것이 좋다.
- 문제 해결은 구체적 지시, 비율 재확인(16:9 vs 9:16), 최종본 해상도 상향, 지연 시 4K/네트워크/부하 점검이 포인트다.

## 실습/실행 단계
1. `/start-3-1-2`로 학습 모듈을 시작한다.
2. 자연어로 첫 이미지를 생성해 기본 결과를 확인한다.
3. 필요하면 `aspect_ratio`와 `resolution`을 명시해 재생성한다.
4. 한 번에 한 가지씩 변경 요청하며 반복 개선한다.
5. 방향이 크게 바뀌면 새 session으로 전환한다.
6. 결과가 확정되면 2K로 다시 생성해 최종본을 만든다.

## 기억할 포인트
- 파라미터를 외우기보다 자연어 지시와 반복 개선이 더 중요하다.
- 초반 1K 반복이 시간 대비 효율이 가장 높다.
- 모호한 표현보다 수정 항목을 명확히 말해야 반영이 잘 된다.
- 좋은 초안을 버리기보다 점진적으로 다듬는 편이 성공 확률이 높다.
