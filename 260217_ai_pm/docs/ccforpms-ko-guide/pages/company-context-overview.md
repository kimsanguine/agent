# TaskFlow 회사 개요
원문: https://ccforpms.com/company-context/overview

## 한 줄 요약
TaskFlow는 원격 협업 중심 PM SaaS를 가정한 코스용 회사이며, 2025년 핵심 과제는 Mobile·Enterprise·Activation 개선이다.

## 핵심 내용
- 코스 전반에서 사용하는 가상 회사 맥락으로, 2021년 설립·Series B·약 50명 조직 설정이다.
- 현재 지표는 50,000 가입자, 10,000 MAU, 2,500 DAU, $2.5M ARR, $250k MRR, 유료 팀 500개다.
- 성장률은 사용자 MoM 15%, 매출 MoM 20%지만 net retention 65%, activation 45%가 주요 개선 포인트다.
- 주 타깃은 Series A-C(50~500명) 원격/하이브리드 팀이며, 이후 Enterprise(500+)로 확장하려 한다.
- 경쟁 구도에서 Asana/Linear/Monday.com/ClickUp 대비 단순성, 속도, PM 친화성, 가격 경쟁력을 내세운다.
- 제품 철학은 async-first, context-rich task, 빠른 성능, 실용적이면서도 완성도 높은 UI에 집중한다.
- 2025 우선순위는 iOS/Android 앱 출시, Enterprise readiness(SSO·권한·audit log), onboarding 강화로 activation 60% 달성이다.
- Senior PM 역할은 onboarding, 핵심 task flow, notifications, integrations(Slack, GitHub 등)이며 핵심 KPI는 activation·time-to-value·viral coefficient다.

## 실습/실행 단계
1. 현재 지표와 2025 OKR 사이의 gap(예: activation 45%→60%, deal size $5k→$15k)을 수치로 정리한다.
2. 담당 영역별 문제를 정의하고 PRD 가설(온보딩 개선, 첫 task 완료 단축, 초대 전환 개선)을 작성한다.
3. Product/Engineering/Design/Mobile 리더와 함께 우선순위와 실행 범위를 cycle 단위로 합의한다.
4. 릴리스 후 activation, time to first task completed, invites per user를 실험 단위로 추적·개선한다.

## 기억할 포인트
- 이 페이지는 개별 과제의 기준 맥락이므로, 모든 제안은 회사 목표·지표와 연결해야 한다.
- TaskFlow의 승부처는 기능 과잉이 아니라 비동기 협업 효율, 맥락 전달, 속도, 사용성이다.
- startup 친화성과 Enterprise 요구사항을 동시에 맞추는 제품 전략 균형이 중요하다.
