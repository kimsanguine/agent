# 4.1: 설정
원문: https://ccforpms.com/vibe-coding/setup

## 한 줄 요약
프로젝트 시작 준비를 하고, AI 협업의 기본 흐름인 "Plan → Build → Iterate → Save → Go Live"를 익히는 레슨이다.

## 핵심 내용
- 약 10분 분량이며, Claude Code에서 `/start-4-1` 실행으로 인터랙티브 학습을 시작한다.
- 이전 모듈 자료를 이미 받았어도, Module 4에는 새 파일이 있어 course files를 다시 다운로드해야 한다.
- Vibecoding은 직접 코드를 치기보다 원하는 결과를 분명히 설명하고 AI가 구현하게 하는 방식이다.
- 이 모듈에서 역할은 PM(요구사항·피드백)과 Claude(구현)로 나뉜다.
- 이 협업 방식은 개발자뿐 아니라 디자이너, 에이전시, 다른 AI 도구와 일할 때도 그대로 적용된다.
- Next.js는 웹 개발용 framework로, Vercel 배포와 잘 맞는다.
- JavaScript는 웹 인터랙션의 핵심 언어지만, 본 과정에서는 Claude가 작성한다.
- 다음 단계인 4.2: Plan에서 Claude가 질문을 통해 빌드 범위를 구체화한다.

## 실습/실행 단계
1. Claude Code에서 `/start-4-1`을 실행한다.
2. Module Overview 안내를 따라 Module 4용 course files를 다시 받는다.
3. 만들고 싶은 결과를 먼저 정리해 Claude에게 구현을 요청한다.
4. 결과를 검토하고 피드백을 주며 반복 개선한다.
5. 완료본을 GitHub에 저장하고 배포(Go Live) 단계로 이어간다.

## 기억할 포인트
- 핵심 역량은 코딩 자체보다 요구사항 명확화와 피드백 품질이다.
- "Describe → Build → Review → Refine" 반복이 품질을 결정한다.
- 기술 세부를 몰라도 협업 루프를 지키면 실전 결과물을 만들 수 있다.
