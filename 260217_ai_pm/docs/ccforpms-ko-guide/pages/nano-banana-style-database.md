# 3.1.4: 스타일 데이터베이스 구축하기
원문: https://ccforpms.com/nano-banana/style-database

## 한 줄 요약
이미지 결과를 일회성으로 끝내지 않고 `style-library.html`에 축적해, 재사용 가능한 스타일 시스템으로 속도와 품질을 함께 높이는 모듈입니다.

## 핵심 내용
- 핵심은 이미지 생성 자체보다 “저장→재사용→축적” 루프를 만드는 메타 스킬입니다.
- 모듈 정보: 약 20분 분량, 선행 학습은 Module 3.1.3, 시작 명령은 `/start-3-1-4`.
- 스타일 라이브러리(`style-library.html`)에는 ID, 썸네일, 이름, 카테고리, 태그, 프롬프트, 활용 예시가 저장됩니다.
- 스타일 적용은 `Generate [your subject] using style #X`처럼 번호 기반으로 호출합니다.
- 라이브러리 확장 방법은 3가지입니다: 작업 중 저장, 온라인 프롬프트 수집/검증, 참고 이미지에서 스타일 추출.
- `style_extract.py`와 Gemini 3 Pro를 이용해 색감, 조명, 구도, 질감, 분위기, 타이포, 효과를 텍스트 스타일로 변환할 수 있습니다.
- 카테고리/태그 정리를 잘하면 일관성 있는 산출물을 빠르게 반복 생산할 수 있습니다.

## 실습/실행 단계
1. Claude Code에서 `/start-3-1-4`를 실행합니다.
2. 이미지 생성 후 만족스러운 결과를 바로 라이브러리에 저장합니다.
3. `style-library.html`을 브라우저에서 열어 카테고리와 태그 기준으로 정리 상태를 확인합니다.
4. 외부 프롬프트는 먼저 테스트 생성 후, 결과가 좋을 때만 저장합니다.
5. 참고 이미지는 `style_extract.py`로 스타일을 추출해 새 주제에 재적용합니다.
6. 필요하면 검색, 컬럼, 다크 모드 등 라이브러리 UI를 커스터마이징합니다.
7. 다음 실전 모듈은 `/start-3-2-1`로 진행합니다.

## 기억할 포인트
- 장기 효과는 스타일 개수보다 재현성과 품질이 검증된 스타일 비중에서 나옵니다.
- 좋은 결과는 즉시 저장해야 누락이 줄고 자산화가 빨라집니다.
- 추출 스타일이 어긋나면 더 단순한 레퍼런스로 재추출하거나 참조 이미지를 함께 사용합니다.
- 저장 반영이 안 보이면 브라우저 새로고침과 저장 완료 여부부터 점검합니다.
