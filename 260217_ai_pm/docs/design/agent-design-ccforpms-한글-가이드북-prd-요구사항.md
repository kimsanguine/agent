# Agent Design — ccforpms-한글-가이드북-prd-요구사항

## 작업 컨텍스트
- 목적: ccforpms.com 내용을 한국 PM 맥락에 맞춰 재구성하기 위한 PRD 요구사항 초안을 수립한다.
- 입력:
  - ccforpms.com 핵심 구조/메시지
  - 기존 `prd.md` 초안/관련 노트
  - 사용자 관점(한국 PM 대상 실무 활용 맥락)
- 출력:
  - `prd.md` 요구사항 초안(Problem/Goal, Scope, FR/NFR, 단계별 실행 항목)
  - 후속 확장(Obsidian 한글 번역본, habix.ai 반영) 작업 정의
- 제약조건:
  - 한국어 중심 문서화
  - 초기 단계에서는 구현보다 요구사항 정의에 집중
  - 한국 PM 대상 가치 제안이 명확해야 함

## 워크플로우
- 단계:
  1. 원본 콘텐츠 구조 파악
  2. 한국 PM 관점 문제/목표 재정의
  3. PRD 요구사항(기능/비기능) 초안 작성
  4. 번역본/웹 반영 후속 작업 분리
- 분기:
  - 콘텐츠 범위 과대 시 MVP 범위 우선 정의
- 실패 처리:
  - 요구사항 모호 시 인터뷰 로그 보강 후 재작성

## 에이전트/스킬 구조
- 메인 역할: PRD 요구사항 작성/정합성 검토
- 서브에이전트: (초안) 리서치, 번역, 정보구조 검토
- 스킬/스크립트: Obsidian 기반 문서 생성/수정
