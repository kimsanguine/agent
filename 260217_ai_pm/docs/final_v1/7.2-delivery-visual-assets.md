# 4.2 비주얼 에셋: 페르소나부터 마케팅 이미지까지

## 🔗 하네스 엔지니어링과의 연결

이 Part는 하네스 엔지니어링의 **작업 구조(Task Structure)**에서 에셋 생성 표준을 정의하는 가이드입니다.

### 연결 지점

- **작업 구조 — 에셋 생성 표준**: 5가지 비주얼 에셋 유형별 생성 규칙이 Application Harness의 표준화된 프로세스입니다.
- **PM 직접 제작**: PM이 디자이너 없이 초기 시각화를 구현하는 Delivery 단계의 자립 원칙을 실현합니다.

**참고**: [dependency-map.md](./dependency-map.md)의 작업 구조(Task Structure) 매핑 섹션을 먼저 읽으면 이 Part의 내용이 더 명확해집니다.

## 시각적 에셋의 가치

제품 개발 과정에서 PM이 필요로 하는 시각적 자산들은:
- **프레젠테이션의 설득력** 증가
- **이해관계자와의 커뮤니케이션** 명확화
- **초기 컨셉 검증** 가속화

Claude Code와 Gemini 이미지 생성 API를 결합하면, 디자이너의 정교화 전에 PM이 직접 초기 시각화를 구현할 수 있습니다.

## PM에게 유용한 5가지 비주얼 에셋

### 1. 페르소나 초상화 (Persona Portraits)

**용도**: 팀 회의, 문서, 프레젠테이션

```
> 우리 제품의 타겟 페르소나 3명에 대한 초상화를 만들어줘.

페르소나 정의:
1. 마리아 (35세, 마케팅 매니저, 서울)
   - 분주한 일정, 데이터 기반 의사결정 중심
   - 스타일: 프로페셔널, 세련됨

2. 준호 (28세, 스타트업 창업자, 판교)
   - 시간 부족, 빠른 피드백 요구
   - 스타일: 캐주얼, 에너지 넘침

3. 수진 (52세, 의료기관 운영자, 대전)
   - 신뢰성 중시, 보수적 접근
   - 스타일: 전문가다운, 따뜻함

Claude: 각 페르소나의 특징을 담은 초상화를 생성합니다.

✓ Junho_Founder.png 생성
✓ Sujin_Healthcare.png 생성

[생성된 이미지 3개]
```

**결과**: PRD, 회의 자료, 전사 공유 문서에 바로 활용 가능

---

### 2. 사용자 여정 맵 시각화 (User Journey Map)

**용도**: 문제점 분석, 이해관계자 정렬

```
> 온보딩 플로우의 사용자 여정을 시각화해줘.

여정 단계:
1. 인지 (Awareness): 광고로 처음 접함
2. 가입 (Sign-up): 이메일 인증
3. 초기화 (Setup): 프로필 정보 입력
4. 첫 사용 (First Action): 첫 번째 작업 완료
5. 재방문 (Retention): 7일 후 재접속

각 단계에서의 감정, 문제점, 해결책을 포함해줘.

Claude: 사용자 여정 맵을 생성합니다.

┌─────────────────────────────────────┐
│     AWARENESS    SIGNUP  SETUP  ... │
├─────────────────────────────────────┤
│ 😊 → 😐 → 😕 → 😊 → 😊           │ (Emotion)
├─────────────────────────────────────┤
│ Pain: 광고 불신   비밀번호   ...  │
│ Solution: 추천글  2FA 간소화   ... │
└─────────────────────────────────────┘

[상세한 시각화 다이어그램 생성]
```

**결과**: 팀 전체의 문제점 공감 능력 향상, 우선순위 결정 근거 제공

---

### 3. 와이어프레임 → 목업 변환 (Wireframe to Mockup)

**용도**: 설계 정밀화, 이해관계자 리뷰

```
> 이전에 만든 성격 유형 테스트의 결과 페이지를
  고해상도 목업으로 변환해줘.

요구사항:
- 결과 카드: 유형명, 색상 코드, 설명
- 특징 섹션: 4가지 주요 특징을 아이콘과 함께 표시
- CTA: "결과 공유", "다시 시도" 버튼
- 배경: 미묘한 그래디언트

Claude: 고해상도 목업을 생성합니다.

✓ Result_Page_Mockup_Desktop.png
✓ Result_Page_Mockup_Mobile.png

[생성된 목업 이미지들]
```

**결과**: 개발팀과의 시각적 정렬, 디자이너의 정교화 시간 단축

  어떻게 보이는지 목업으로 보여줘.

참고: 성격 유형 테스트의 결과 페이지

### 5. 마케팅용 라이프스타일 이미지 (Lifestyle Images)

**용도**: 랜딩페이지, 소셜 미디어, 광고 크리에이티브

```
> 우리 제품을 사용하는 일상적인 라이프스타일 이미지를 만들어줘.

씬 1: 직장인이 업무 중 빠르게 성격 유형 테스트를 수행
     - 배경: 모던 오피스 (자연광)
     - 감정: 집중, 호기심
     - 색감: 밝고 따뜻함

씬 2: 팀원들이 자신의 테스트 결과를 공유하며 웃음
     - 배경: 카페 같은 협업 공간
     - 감정: 즐거움, 유대감
     - 색감: 생동감 있음

Claude: 라이프스타일 이미지를 생성합니다.

✓ Office_Scene_Lifestyle.png
✓ Team_Collaboration_Scene.png

[고품질의 라이프스타일 사진 같은 이미지들]
```

**결과**: 즉시 마케팅 캠페인 활용 가능한 콘텐츠

## 스타일 일관성 유지 팁

### 스타일 가이드 정의

```
> 모든 비주얼 에셋에 공통으로 적용할
  스타일 가이드를 정의해줘.

Claude: 생성됨
```

**스타일 요소:**
- **색상 팔레트**: 주색(Primary) 3~4개, 보조색(Secondary) 2~3개
- **타이포그래피**: 제목(Title), 본문(Body), 캡션(Caption) 폰트 지정
- **일러스트레이션**: 선 굵기, 코너 라운드, 스타일 통일
- **레이아웃**: 마진, 패딩, 그리드 시스템 정의

### 프롬프트에 스타일 포함

모든 요청에 일관된 스타일 설명 추가:

```
> [기본 요청]...

스타일 가이드 적용:
- 색상: Primary Blue (#003D82), Accent Orange (#FF6B35)
- 폰트: Pretendard Bold (제목), Pretendard Regular (본문)
- 일러스트: 미니멀, 둥근 코너, 라인 2px
```

## PM 역할 확장 포인트

### BEFORE: 디자이너 의존
```
PM이 비주얼 에셋 필요
  ↓
디자이너에게 요청
  ↓
디자이너 작업 (2~3일)
  ↓
피드백 및 수정 반복 (1~2일)
  ↓
최종 에셋 완성 (총 3~5일)
```

**문제점**: 초기 컨셉 단계에서 비주얼 검증 불가, 마케팅 자료 준비 지연

---

### AFTER: AI 네이티브 PM
```
PM이 초기 비주얼 콘셉트 직접 생성 (30분)
  ↓
팀 리뷰 및 피드백
  ↓
디자이너는 정교화에만 집중 (1~2일)
  ↓
최종 프리미엄 에셋 완성 (총 2~3일)
```

**이점**:
- **초기 컨셉 검증**: PM이 즉시 시각화하여 방향성 확정
- **디자이너 생산성**: 반복 수정 단계 최소화
- **마케팅 속도**: 임시 콘텐츠를 즉시 활용 가능
- **이해관계자 정렬**: 구체적 시각 자료로 논의 명확화

**시간 절감**: 3~5일 → 2~3일 (30% 단축)
**PM의 새로운 책임**: 초기 시각 방향성 정의 및 빠른 반복
