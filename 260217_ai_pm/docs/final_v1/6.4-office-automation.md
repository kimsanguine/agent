# 6.4 Office 자동화: PowerPoint & Excel (2026년)

## 개요

2026년 Anthropic의 가장 혁신적인 기능 중 하나는 **Claude in PowerPoint**와 **Claude in Excel**이다. 이제 PM이 수동으로 하는 리포트 작업의 80%가 자동화될 수 있다.

---

## Part A: Claude in PowerPoint (02/05 출시)

### 기존 vs 새로운

#### 기존 방식 (2025년)
```
PM이 데이터 수집 (1시간)
   ↓
슬라이드 템플릿 열기 (10분)
   ↓
손으로 그래프 입력 (3시간)
   ↓
텍스트 작성 (2시간)
   ↓
디자인 조정 (1시간)
━━━━━━━━
총: 7시간 (50장 슬라이드)
```

#### 새로운 방식 (2026년)
```
PM이 데이터 파일 준비 (20분)
   ↓
/ppt-from-data quarterly_metrics.csv
   ↓
Claude가 50장 슬라이드 자동 생성 (3분)
━━━━━━━━
총: 23분 (기존 대비 95% 단축)

+ 생성된 슬라이드 UI에서 1클릭 수정 가능
```

---

### 사용법

#### 기본 커맨드

```bash
# 기본: 데이터 파일 → 슬라이드 자동 생성
/ppt-from-data quarterly_metrics.csv

# 고급: 특정 템플릿 사용
/ppt-from-data quarterly_metrics.csv \
  --template "investor-pitch" \
  --slides 30 \
  --style "minimalist"

# 더 고급: 분석 결과를 슬라이드로
/ppt-from-analysis --input performance-report.json \
  --include-charts \
  --include-recommendations
```

#### 생성되는 슬라이드 구조

```
슬라이드 1: 제목 (자동)
슬라이드 2: Executive Summary (자동)
슬라이드 3-5: 지표 분석 (자동 생성 차트)
슬라이드 6-8: 시각화 및 인사이트 (자동)
슬라이드 9: 권장사항 (자동 도출)
슬라이드 10: 다음 스텝 (자동)

총 10-50장 (데이터 규모에 따라)
```

---

### 실제 사용 사례

#### 사례 1: 분기별 경영진 리포트

**상황**:
- 분기마다 CEO에게 보고할 50장 슬라이드 필요
- 기존: 3일 소요 (48시간)
- 새로운: 30분 소요

**프로세스**:

```
1단계: 데이터 수집 (20분)
- 매출: monthly_revenue.csv
- 지표: kpi_dashboard.json
- 고객: customer_metrics.csv
- 제품: product_roadmap.md

2단계: Claude로 슬라이드 생성 (3분)
/ppt-quarterly \
  --revenue monthly_revenue.csv \
  --kpi kpi_dashboard.json \
  --customers customer_metrics.csv \
  --roadmap product_roadmap.md \
  --tone "executive"

3단계: UI에서 미세 조정 (7분)
- 슬라이드 순서 재배열 (드래그)
- CEO 개인화 메시지 추가 (1쿨릭 텍스트 편집)
- 강조 색상 변경 (테마 적용)

3단계: 팀 공유 (1분)
/ppt-share quarterly-Q3-2026.pptx \
  --send-to "ceo@company.com" \
  --cc "leadership-team" \
  --notify-slack
```

**효과**:
- 시간: 48시간 → 30분 (96% 단축)
- 비용: CEO 시간 절감 ($2,000 가치)
- 품질: 일관성 향상, 실수 감소

---

#### 사례 2: 주간 팀 리포트 (완전 자동화)

**설정**:

```bash
# 매주 금요일 15:00 자동 생성
/loop 15:00 "Friday" /ppt-weekly-report \
  --data weekly-metrics.json \
  --include-blockers \
  --include-wins \
  --include-next-week-priorities

# 생성 후 자동 공유
--post-to slack \
  --channel "#team-updates" \
  --message "주간 리포트 준비됨! [보기]"
```

**결과**:
- 매주 자동 생성 (PM 개입 0)
- 팀이 금요일 오후 4시에 확인 가능
- 시간: 월 4시간 절감

---

### 고급 기능: Charts & Diagrams (03/13)

#### 자동 생성 차트

Claude가 데이터를 분석해 **최적의 차트 자동 선택**:

```
데이터 입력:
month, revenue, growth
Jan, $1M, 5%
Feb, $1.2M, 20%
Mar, $1.5M, 25%

   ↓

Claude 분석:
"이 데이터는 추세를 보여주므로 꺾은선 차트가 최적"

   ↓

자동 생성:
[꺾은선 차트 with 색상, 범례, 레이블]
```

#### 자동 생성 Diagram

```bash
/diagram-from-text \
  --input "마케팅 프로세스: 아이디어 → 컨셉 → 디자인 → 배포 → 분석" \
  --style "flowchart"

   ↓

[자동으로 flowchart 생성]
```

## Part B: Claude in Excel (02/05 출시)

### 기존 vs 새로운

#### 기존 방식 (2025년)
```
재무 모델 템플릿 열기
   ↓
수식 수동 입력 (2시간)
   ↓
데이터 입력 (1시간)
   ↓
다중 시나리오 수동 입력 (3시간)
   ↓
검증 (1시간)
━━━━━━━━
총: 7시간
```

#### 새로운 방식 (2026년)
```
데이터 파일 준비 (20분)
   ↓
/excel-forecast --model financial.csv
   ↓
Claude가 모든 수식 자동 생성 (1분)
   ↓
시나리오 분석 자동 생성 (1분)
━━━━━━━━
총: 22분 (기존 대비 95% 단축)
```

---

### 사용법

#### 기본 커맨드

```bash
# 기본: 재무 예측 모델 자동 생성
/excel-forecast financial_2026.csv

# 시나리오 분석: 낙관적/중간/보수적
/excel-scenarios \
  --base financial_2026.csv \
  --scenarios optimistic,base,conservative

# 동적 업데이트: 매월 자동 재계산
/excel-auto-update financial_2026.csv \
  --frequency monthly \
  --on "first-day-of-month" \
  --notify-slack
```

---

### 실제 사용 사례

#### 사례 1: 분기별 재무 예측

**상황**:
- 매분기마다 12개월 예측 필요
- 기존: 매번 3시간 소요
- 새로운: 10분 소요

**프로세스**:

```
1단계: 데이터 수집 (5분)
- 역사적 매출: sales_history.csv
- 비용: cost_structure.csv
- 성장률: growth_assumptions.csv

2단계: Claude로 모델 자동 생성 (2분)
/excel-quarterly-forecast \
  --sales sales_history.csv \
  --costs cost_structure.csv \
  --growth growth_assumptions.csv \
  --months 12 \
  --scenarios 3

3단계: 검증 및 미세 조정 (3분)
- 수식 논리 확인 (자동 설명)
- 가정 수정 필요하면 1클릭 변경
- 즉시 모든 셀 재계산

4단계: 경영진과 공유 (1분)
/excel-share forecast-Q4-2026.xlsx \
  --recipients "cfo@company.com, ceo@company.com" \
  --permissions "read-only" \
  --notify-slack
```

**효과**:
- 시간: 3시간 → 11분 (94% 단축)
- 정확성: 자동 수식 검증으로 오류 0%
- 유연성: 가정 변경 시 자동으로 모든 시나리오 업데이트

---

#### 사례 2: 예산 최적화 분석 (실시간)

**설정**:

```bash
# 월간 예산 파일을 자동으로 분석
/excel-budget-analysis \
  --input monthly_budget.csv \
  --frequency monthly \
  --analyze-variance \
  --suggest-reallocations \
  --post-report slack

결과 보고서 자동 생성:
├─ 예산 대비 실적 (예)
├─ 분산 분석 (원인)
├─ 최적화 제안 (어디에 돈을 쓸지)
└─ 다음 달 권장 예산
```

**자동화**:
```
매월 1일 자동 실행
   ↓
지난달 예산 데이터 수집
   ↓
Claude 분석 및 보고서 생성
   ↓
CFO에게 Slack 알림 + 공유
   ↓
CFO: "좋아, 이 제안대로 해"
   ↓
자동으로 다음달 예산 업데이트
```

**효과**:
- 시간: 월 8시간 절감
- 의사결정 개선: 데이터 기반 (추측이 아님)
- 비용 최적화: 분석에 기반한 재배치

### 통합 자동화: Data → Slide → Share

```
1단계: 데이터 생성
━━━━━━━━━━━━━━━━
여러 도구에서 데이터 수집:
├─ Google Sheets (마케팅 지표)
├─ Jira (개발 진행률)
├─ Stripe (매출)
└─ 기타 도구들

2단계: Claude로 통합 분석
━━━━━━━━━━━━━━━━
/analyze-all-data \
  --sources "sheets, jira, stripe, ..." \
  --format json

3단계: PowerPoint 자동 생성
━━━━━━━━━━━━━━━━
/ppt-from-analysis \
  --input analysis-output.json \
  --create-charts \
  --create-diagrams

4단계: Excel 예측 모델 추가
━━━━━━━━━━━━━━━━
/excel-forecast \
  --based-on analysis \
  --include-scenarios

5단계: 통합 공유
━━━━━━━━━━━━━━━━
/share-all-reports \
  --ppt quarterly-report.pptx \
  --excel forecast.xlsx \
  --recipients "leadership@company" \
  --notify-slack
```

**전체 시간**: 30분 (기존: 2일)

## Part E: PM 의사결정 가이드

### 언제 PowerPoint 자동화를 쓸까?

✅ **적극 권장**:
- 분기별 경영진 리포트 (매번 50장+)
- 주간/월간 팀 업데이트 (반복)
- 투자자 피치 (자주 업데이트)
- 고객 프레젠테이션 (데이터 기반)

❌ **추천하지 않음**:
- 크리에이티브 프레젠테이션 (브랜드 스토리)
- 한 번의 특별한 회의 (비용 대비 효과 낮음)

### 언제 Excel 자동화를 쓸까?

✅ **적극 권장**:
- 재무 예측 모델 (매월)
- 예산 분석 (매월)
- KPI 대시보드 (매일/매주)
- 시나리오 플래닝 (반복)

❌ **추천하지 않음**:
- 일회성 분석
- 커스텀 계산이 매우 복잡한 경우
