# 6.5 Computer Use: 마우스와 키보드 자동화 (2026년)

## 🔗 하네스 엔지니어링과의 연결

이 Part는 하네스 엔지니어링의 **실행 루프(Execution Loop)**에서 UI 기반 자동화를 구현하는 가이드입니다.

### 연결 지점

- **실행 루프 — UI 기반 자동화**: Claude의 마우스/키보드 제어 능력으로 브라우저, 설계 도구, 업무 앱을 직접 조작하는 새로운 실행 루프입니다.
- **Harness 확장**: API가 없는 도구도 Harness의 실행 루프에 포함시킬 수 있게 합니다.

**참고**: [dependency-map.md](./dependency-map.md)의 실행 루프(Execution Loop) 매핑 섹션을 먼저 읽으면 이 Part의 내용이 더 명확해집니다.

## 개요

2026년 3월부터 Claude는 **마우스와 키보드를 직접 제어**할 수 있습니다. 브라우저, 설계 도구, 업무 앱을 Claude가 조작합니다. UI 자동화의 신세계입니다.

## Part A: Computer Use 개념

### 무엇을 할 수 있나?

```

└─ 파일 탐색기 (폴더 정렬, 배치 이름 변경)
```

## Part B: 기존 vs 새로운 방식

### Before: 수동 UI 조작 (2025년)

```
PM의 일상:
Jira 티켓 생성 (반복)
  1. Jira 열기
  2. "Create Issue" 클릭
  3. Title 입력 (복사-붙여넣기)
  4. Description 입력 (템플릿 사용)
  5. Priority, Label, Assignee 선택
  6. 제출
  ─────────────────
  소요: 5-10분/건 × 10건 = 50-100분/주
```

### After: Claude Computer Use (2026년)

```
PM의 일상:
/auto-jira-batch 1월_sprint.csv
   ↓
Claude가:
  1. Jira 자동 열기
  2. 매 행마다 "Create Issue" 자동 클릭
  3. 데이터 파일에서 자동 입력
  4. 규칙 기반으로 Priority/Label 자동 선택
  5. 팀원에게 자동 할당
  6. 완료
  ─────────────────
  소요: 2분 (98건 자동 생성)
```

**효과: 50-100분 → 2분 (95% 시간 절감)**

## Part C: 사용법

### 기본 명령어

```bash
# 1. 단순 UI 자동화
/computer-use --task "Jira에 새 버그 티켓 3개 만들어. 제목은: \
  [Bug] API latency (P1, 개발팀 할당) \
  [Bug] Login 버튼 미표시 (P2, QA 할당) \
  [Bug] 스페인어 번역 오류 (P3, 마케팅 할당)"

# 2. 파일 기반 자동화
/computer-use-batch --file marketing_tasks.csv \
  --action "Google Forms에 각 행을 제출"

# 3. 정확도 요구
/computer-use --task "..." \
  --accuracy-target 98 \
  --retry-on-failure
```

## Part D: 실제 사용 사례

### 사례 1: Jira 티켓 배치 생성

**상황:**
- 월간 스프린트 계획: 25-30개 티켓 생성
- 기존: 팀 리드가 수동으로 2-3시간 소요
- 새로운: Claude가 자동으로

**프로세스:**

```
1단계: 데이터 준비 (10분)
──────────────────
Sprint_202603.csv:
title | description | priority | assignee | epic
------|-------------|----------|----------|------
API latency | ... | P1 | dev@... | Performance
...

2단계: Claude 자동화 명령 (1분)
──────────────────
/computer-use-batch \
  --file Sprint_202603.csv \
  --app jira \
  --action "create-issue" \
  --team-mapping "dev@company.com -> Team Dev"

3단계: 자동 실행 (3분)
──────────────────
Claude:
  • Jira 자동 로그인
  • 25개 행 반복 처리
  • 각 행 → Jira 티켓 자동 생성
  • 담당자 자동 할당
  • 완료 알림 (Slack으로)

결과:
  • 25개 티켓 생성 완료 (95% 성공률)
  • 1개 실패 (수동 재시도)
  • 총 4분 소요
```

**효과:**
- 시간: 2-3시간 → 4분 (94% 절감)
- 인력: 팀 리드 → 자동화
- 정확도: 98% (수동: 85%)

### 사례 2: Figma 배치 색상 변경

**상황:**
- 디자인 시스템 컬러 업데이트
- 50개 컴포넌트의 배경색을 #FF5733 → #3B82F6 로 변경
- 기존: 디자이너가 50번 수동 클릭 (1-2시간)
- 새로운: Claude 자동화

**프로세스:**

```
1단계: 작업 정의 (2분)
──────────────────
/computer-use \
  --app figma \
  --task "
    Figma 열기 (Design System 파일)
    각 컴포넌트:
      1. 선택
      2. Fill 색상을 #FF5733에서 #3B82F6로 변경
      3. 저장
    50개 컴포넌트 모두 완료"

2단계: 자동 실행 (15분)
──────────────────
Claude가 반복:
  1. Figma 인터페이스 읽기 (OCR)
  2. 색상이 #FF5733인 컴포넌트 식별
  3. 클릭해서 선택
  4. 색상 패널 열기
  5. 새 색상 #3B82F6 입력
  6. 다음 컴포넌트로

3단계: 결과 확인 (3분)
──────────────────
완료 보고:
  ✓ 48/50 성공 (96%)
  ⚠️ 2개 실패 (텍스트 색상 - 규칙 오류)
```

**효과:**
- 시간: 1-2시간 → 18분 (85% 절감)
- 정확도: 96%
- 반복 가능: 다음 색상 업데이트도 같은 스크립트 재사용

### 사례 3: Google Forms 자동 응답 (고객 조사)

**상황:**
- 주간 고객 만족도 조사 설문지 제출
- 내부 템플릿이 있으나 매주 3-4명의 팀이 수동으로 제출
- 기존: 각 팀 대표가 양식 열기 → 입력 → 제출 (5분 × 4팀 = 20분)
- 새로운: Claude 자동화

**프로세스:**

```
1단계: 데이터 파일 준비 (5분)
──────────────────
feedback.json:
{
  "teams": [
    {
      "name": "Engineering",
      "lead": "alice@",
      "satisfaction": 8,
      "comment": "열정적 팀"
    },
    ...
  ]
}

2단계: 자동화 스크립트 (1분)
──────────────────
/computer-use-batch \
  --file feedback.json \
  --form "https://forms.google.com/..." \
  --field-mapping "
    name -> Team
    satisfaction -> Satisfaction (1-10)
    comment -> Comments"

3단계: 자동 제출 (2분)
──────────────────
Claude:
  1. Google Forms URL 열기
  2. 각 팀 데이터 읽기
  3. 필드에 자동 입력
  4. "Submit" 클릭
  5. 다음 팀으로
  6. 4개 양식 모두 제출 완료

결과: 4건 제출 완료 (100% 성공)
```

**효과:**
- 시간: 20분 → 3분 (85% 절감)
- 오류: 0건 (자동화이므로 입력 오류 없음)
- 재현성: 매주 같은 스크립트로 반복

## Part E: 고급 사용법

### 1. 재시도 로직

실패했을 때 자동 재시도:

```bash
/computer-use \
  --task "Jira에 티켓 생성" \
  --max-retries 3 \
  --retry-delay 5 \
  --on-failure "slack #devops"
```

### 2. 조건부 로직

상황에 따라 다른 작업:

```bash
/computer-use \
  --task "
    Jira 열기
    IF 티켓이 이미 있으면:
      → 업데이트
    ELSE:
      → 새로 생성" \
  --condition-check "query:/jira/search?jql=..."
```

### 3. 크로스 앱 워크플로우

여러 앱을 연결:

```bash
/computer-use-workflow \
  --steps "
    1. Google Sheets에서 고객 데이터 읽기
    2. Jira에 각 고객별 티켓 생성
    3. Slack에 생성 완료 알림
    4. CSV 파일로 로그 저장"
```

## Part F: 성능 vs 정확도 트레이드오프

### 정확도별 소요 시간

```
빠른 처리 (80% 정확도)
  • 약간의 실패 허용
  • 수동 검증 없음
  • 예: 색상 변경, 대량 복사

표준 처리 (94% 정확도)
  • 기본값 (Sonnet 4.6)
  • 대부분의 작업 적합
  • 예: 티켓 생성, 양식 제출

정확한 처리 (98% 정확도)
  • 느림 (더 많은 재시도)
  • 중요한 작업만
  • 예: 금융 거래, 법무 문서
```

### 추천 사용 시나리오

| 작업 | 정확도 | 재시도 |
|------|--------|--------|
| 대량 입력 (배경색 변경) | 90% | 1회 |
| 중요 티켓 (버그 생성) | 95% | 2회 |
| 민감한 작업 (급여 입력) | 98% | 3회 |
| 테스트 (UI 검증) | 85% | 0회 |

## Part G: 비용 vs 효과 분석

### 시간 절감 계산

```
예시: Jira 티켓 배치 생성

월간 작업:
  • 스프린트 당 25개 티켓 (월 2회 스프린트 = 50개)
  • 팀 리드가 2시간 소요 × 2회 = 월 4시간

Claude Computer Use 적용:
  • 1회 설정: 10분 (스크립트 작성)
  • 1회 실행: 4분 × 2회 = 8분/월
  • 유지보수: 5분/월

절감:
  • 기존: 240분/월
  • 신규: 25분/월
  • 절감: 215분/월 (89% 감소)
  • 연간: 2,580분 = 43시간 절감
  • 비용: 팀 리드 $60/시간 × 43시간 = $2,580 절감/연
```

### ROI 계산

```
투자:
  • Claude API (Computer Use): $0.05/분 × 50분/월 × 12 = $30/연
  • 초기 설정: 10시간 × $80/시간 = $800 (1회)

수익:
  • 인력 절감: $2,580/연
  • 품질 향상 (오류 감소): $500/연 추정
  • 처리 속도 향상: $300/연 추정

ROI = ($2,580 + $500 + $300 - $30 - $800) / $830 = 290% (1년 내)
```

## Part H: 한계 & 주의사항

### ❌ 할 수 없는 것

```
❌ 실시간 게임 플레이 (반응 시간 필요)
❌ 영상/음성 편집 (픽셀 단위 제어 어려움)
❌ 매우 복잡한 설계 작업 (수십 개 디테일)
❌ 보안 인증 (2FA 완벽 지원 미약)
❌ 독점 앱 (API 미지원, 특수 프로토콜)
```

### ⚠️ 주의할 점

```
⚠️ 금융/법무 작업은 100% 검증 필수
⚠️ 고객 데이터 포함 작업은 개인정보보호법 확인
⚠️ 빠른 변경되는 UI는 스크립트 유지보수 필요
⚠️ 월 5,000회 이상 자동화는 성능 저하 가능
⚠️ 중단/일시정지 기능 없음 (재시작만 가능)
```
