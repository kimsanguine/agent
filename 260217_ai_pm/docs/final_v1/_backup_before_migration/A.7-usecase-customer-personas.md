# 유즈케이스: 데이터 기반 고객 페르소나 구축

> **PM 역량**: 고객 이해 / 세그멘테이션 / 제품 전략 기반
> **난이도**: P → L
> **관련 문서**: [5.1-discovery-user-research.md](./5.1-discovery-user-research.md), [6.2-definition-product-strategy.md](./6.2-definition-product-strategy.md)

---

## 왜 PM에게 필요한가?

많은 팀이 페르소나를 "상상으로 만든 가상의 인물"로 취급합니다. 이름을 붙이고, 사진을 넣고, "30대 마케터 미나"같은 프로필을 만들지만, 실제 고객 데이터와 연결되지 않아 의사결정에 활용되지 못합니다.

Claude Code는 **실제 데이터(사용 로그, 지원 티켓, 리뷰, 인터뷰 기록)에서 패턴을 추출**하여 데이터 기반 페르소나를 생성합니다. "이 페르소나가 전체 사용자의 몇 %를 차지하고, 어떤 행동 패턴을 보이며, LTV가 얼마인지"를 수치로 보여줍니다.

---

## 상황

당신은 AI Avatar 서비스의 PM입니다. 다음 분기 제품 전략을 수립하기 위해 "누가 우리 제품을 가장 많이 쓰고, 왜 쓰는지"를 체계적으로 파악해야 합니다. 현재 보유한 데이터:

- **사용 로그**: 최근 3개월 활성 사용자 약 12,000명의 행동 데이터 (CSV)
- **지원 티켓**: Zendesk에서 최근 3개월 티켓 1,200건
- **NPS 코멘트**: 최근 분기 NPS 서베이 응답 800건
- **세일즈 CRM**: Hubspot에서 Win/Lose 딜 데이터

---

## 기존 방식 vs Claude 방식

```
기존 (2~3주):
1. 데이터팀에 행동 로그 분석 요청 (3~5일 대기)
2. 지원 티켓 수동 분류 (2일)
3. NPS 코멘트 수동 읽기 + 태깅 (2일)
4. 인사이트 종합 + 페르소나 초안 (2일)
5. 디자인팀에 시각화 요청 (3일)
6. 팀 워크숍에서 검증 (1일)

Claude (2~3일):
1. 행동 로그 클러스터링 + 지원/NPS 패턴 매칭 (3시간)
2. 페르소나 초안 + 데이터 근거 (2시간)
3. PM 검증 + 인터뷰 보강 (1일)
4. 팀 워크숍에서 최종 확정 (반일)
```

## 워크스루

### Step 1: 행동 데이터 클러스터링

```
Terminal 입력:
> 사용자 행동 데이터를 분석해서 자연스러운 세그먼트를 찾아줘.

  데이터: ./data/user-behavior-q4.csv
  컬럼: user_id, signup_date, plan_type, monthly_avatar_count,
        avg_video_length, languages_used, feature_usage_map,
        last_active, session_count, referral_source

  분석 방법:
  1. 주요 행동 변수로 클러스터링 (K-means 또는 적합한 방법)
  2. 각 클러스터의 특성 요약
  3. 클러스터별 LTV, 이탈률, 활성도 비교
  4. 최적 클러스터 수는 데이터가 말하게 해 (3~7개)

  주의: 클러스터링 결과는 "시작점"이야.
  나중에 정성 데이터로 보강할 거니까 해석을 풍부하게 해줘.

Claude Code 응답:
데이터를 로드하고 분석합니다...

✅ 12,847명 로드 (결측값 처리: 312건 제외)
📊 주요 행동 변수 5개로 클러스터링 진행

실루엣 분석 결과 최적 K=4

### Step 3: 페르소나 문서 생성

```
Terminal 입력:
> 클러스터링 + 정성 데이터를 종합해서 4개 페르소나 문서를 만들어줘.

  각 페르소나에 포함할 내용:
  1. 이름과 한 줄 설명 (데이터 기반으로 자연스럽게)
  2. 데이터 프로필 (정량)
  3. Jobs to Be Done (데이터에서 추출)
  4. 페인 포인트 (지원 티켓 + NPS에서)
  5. 기회 영역 (이 페르소나를 더 만족시키려면?)
  6. 제품 전략 시사점

  중요: 각 항목마다 [근거 데이터]를 명시해줘.
  "상상으로 만든 페르소나"가 아니라 "데이터가 말하는 페르소나"여야 해.

Claude Code 응답:

# 📋 AI Avatar 고객 페르소나 (데이터 기반)
## 생성일: 2025-02-21 | 데이터 기간: 2024 Q4

---

## Persona 1: 콘텐츠 팩토리 "팩토리 민수"
> "매달 수백 개의 아바타 영상을 만들어야 해요.
>  자동화와 일관된 품질이 생명입니다."

### 데이터 프로필
| 지표 | 수치 | 근거 |
|------|------|------|
| 비중 | 23% (2,882명) | K-means 클러스터 1 |
| 월 생성량 | 45개 | 행동 로그 평균 |
| 주 플랜 | Business 72% | 플랜 분포 |
| LTV | $2,840 | 코호트 분석 |
| NPS | 52 | NPS 매칭 데이터 |

### Jobs to Be Done
1. 다국어 마케팅 영상을 대량 생성해서 글로벌 캠페인 운영
   [근거: 평균 4.7개 언어 사용, API 배치 호출 패턴]
2. 일관된 브랜드 톤을 유지하면서 빠르게 콘텐츠 생산
   [근거: 커스텀 아바타 사용률 89%, 템플릿 재사용률 높음]
3. 콘텐츠 제작 프로세스를 자동화해서 인력 비용 절감

### 페인 포인트
1. 🔴 배치 생성 시 1~2개 품질 편차 → 전체 재실행 필요
   [근거: 지원 티켓 24%, NPS "불안정" 키워드]
2. 🟡 API Rate Limit에 자주 도달
   [근거: 지원 티켓 32%]
3. 🟡 다국어 립싱크 정확도 언어별 편차
   [근거: 지원 티켓 15%, NPS "정확도" 부정 코멘트]

### 기회 영역
- 배치 품질 보증 기능 (이상 감지 → 자동 재생성)
- API Rate Limit 상향 플랜
- 언어별 립싱크 품질 대시보드

### 제품 전략 시사점
이 페르소나가 전체 매출의 **38%**를 차지합니다.
[근거: LTV $2,840 × 2,882명 / 전체 매출]
→ 유지(Retention) 최우선: 배치 품질 안정화가 핵심
→ 확장(Expansion): Rate Limit 상향 → 자연스러운 업그레이드 경로

⚠️ PM 확인 필요: "팩토리 민수"의 실제 직무를 확인하세요.
   데이터상 에이전시 마케터 또는 내부 콘텐츠팀으로 추정되지만,
   5명 정도 인터뷰로 검증하면 페르소나가 훨씬 풍부해집니다.

## PM 판단 포인트

### ❓ "데이터 기반 페르소나와 전통적 페르소나의 차이는?"

```
전통적 페르소나 (Assumption-based):
- 팀 워크숍에서 가설로 생성
- "우리 고객은 아마 이럴 것이다"
- 시각적으로 예쁘지만 근거 약함
- 의사결정에 거의 활용되지 않음

데이터 기반 페르소나 (Evidence-based):
- 실제 행동 데이터에서 클러스터 도출
- "데이터가 이렇게 말하고 있다"
- 각 항목에 [근거]가 붙어 있음
- 제품 전략, 로드맵 우선순위에 직접 활용

PM이 추가해야 하는 것:
- [인터뷰 검증] 클러스터별 3~5명 인터뷰로 "왜"를 보강
- [비즈니스 맥락] "이 페르소나를 우선할지"는 전략적 판단
- [사용처 정의] 이 페르소나를 어디에 활용할지 결정
  → 로드맵 우선순위? GTM 전략? 세일즈 타겟팅?
```

### ❓ "클러스터가 4개인데, 실제로는 더 세분화해야 하는 건 아닌가?"

```
판단 기준:
1. [액셔너블한가?] 각 페르소나별로 다른 전략을 세울 수 있으면 적정
   → 4개 페르소나에 4가지 다른 액션이 나오면 OK
   → 7개로 늘려도 비슷한 액션이면 오히려 혼란

2. [팀이 기억할 수 있는가?] 페르소나가 5개를 넘으면
   팀원들이 일상에서 사용하지 않음

3. [데이터가 뒷받침하는가?] 실루엣 점수가 급격히 떨어지면
   과분할된 것

→ 4개로 시작하고, 분기별로 데이터를 다시 돌려서 검증하세요.
```

## 실습 과제

### Level 1 (J): 정성 데이터 기반 페르소나
NPS 코멘트나 지원 티켓 데이터(최소 100건)로 2~3개 페르소나를 추출하세요. 각 페르소나에 데이터 근거를 명시하세요.

### Level 2 (P): 정량 + 정성 통합 페르소나
행동 로그 + 정성 데이터를 결합하여 클러스터링 기반 페르소나를 생성하세요. 클러스터별 3명 인터뷰를 수행하여 "왜"를 보강하세요.

### Level 3 (L): 페르소나 기반 제품 전략
페르소나를 활용하여 다음 분기 로드맵 우선순위를 결정하세요. "어떤 페르소나의 어떤 JTBD를 우선하는가"를 데이터와 전략 논거로 문서화하세요.
