# 5.2 KPI 대시보드 & 자동화

## OMTM 중심의 성장 전략

**OMTM (One Metric That Matters)**은 현 단계에서 가장 중요한 **단 하나의 지표**입니다.

이는:
- 모든 팀의 초점을 통일
- 우선순위 갈등 해소
- 명확한 성공/실패 판단

예시:
- 초기 단계: Monthly Active Users (MAU)
- 성장 단계: Activation Rate (가입 → 첫 활동)
- 성숙 단계: Net Revenue Retention (NRR)
- 마지막 단계: Rule of 40 (성장률 + 수익성)

---

## 실전: KPI 체계 설계

### 1단계: Claude와 함께 지표 정의

```
> 우리 제품의 KPI 체계를 설계해줘.

  제품: 팀 협업 도구 (초기 단계)

  요구사항:
  - OMTM 선정 (가장 중요한 지표 1개)
  - 보조 KPI 4~5개 (OMTM 달성을 위한 세부 지표)
  - 각 지표의 정의, 계산식, 측정 주기, 담당자
  - 목표값 (월간, 분기별)
```

### Claude의 KPI 설계 결과

```
# 팀 협업 도구 - KPI 체계

## 📊 OMTM: Weekly Active Teams (WAT)

### 정의
- 최소 1회 이상 활동한 팀의 수
- 활동: 문서 작성, 댓글 추가, 초대

### 계산식
```
WAT = COUNT(DISTINCT team_id
           WHERE activity_count >= 1
           AND week = CURRENT_WEEK)
```

### 측정 주기
- 매일 자정 UTC 기준
- 대시보드 업데이트: 매일 8:00 KST

### 담당자
- Owner: Growth PM (분석 및 전략)
- Reporter: Data Team (자동화)

### 목표
| 기간 | 목표 | 현재 | 진행률 |
|------|------|------|--------|
| 2월 | 2,500팀 | 1,847팀 | 74% |
| 3월 | 3,500팀 | — | — |
| 2026 연간 | 25,000팀 | — | — |

### 주의사항
- 유료 고객만 포함 (트라이얼 제외)
- 비활성 팀(30일 미활동) 제외
- 휴일 및 주말 변동성 고려

## 보조 KPI

### 1️⃣ Activation Rate (가입 → 첫 활동)

```
정의: WAT / 신규 팀(가입 후 7일 내)

계산식:
Activation = COUNT(active_teams) / COUNT(signup_teams in 7d)

목표: 60% (현재: 48%)

측정: 주 1회 (월요일)

담당자: Growth PM
```

### 2️⃣ Net Revenue Retention (NRR)

```
정의: (현월 수익 + 확장 + 업셀) / 전월 수익

계산식:
NRR = (MRR_current + Expansion + Upsell) / MRR_previous

목표: 110% (현재: 95%)

측정: 매월 말일

담당자: Revenue Operations
```

### 3️⃣ Time-to-Value (TTV)

```
정의: 가입부터 첫 번째 문서 작성까지 소요 시간

계산식:
TTV = (timestamp_first_doc_creation - timestamp_signup)

목표: 평균 12분 이내 (현재: 18분)

측정: 매일 자정 (일일 코호트 분석)

담당자: Product Manager
```

### 4️⃣ Feature Adoption Rate

```
정의: 각 주요 기능의 사용률

기능별:
- Comments: 72% (목표: 85%)
- Templates: 45% (목표: 70%)
- Integrations: 38% (목표: 60%)

측정: 주 1회

담당자: Product Analytics
```

### 5️⃣ Churn Rate

```
정의: 월간 이탈 팀 비율

계산식:
Churn = Churned_Teams / Active_Teams_Start_of_Month

목표: < 5% (현재: 7.2%)

측정: 매월 말일

담당자: Customer Success + Product
```

### calculate_kpis.py (예시)

```python
import pandas as pd
from datetime import datetime, timedelta
import requests  # Slack 알림용

# 데이터 로드
teams = pd.read_sql("SELECT * FROM teams", conn)
activities = pd.read_sql("SELECT * FROM activities", conn)

# WAT 계산
current_week = datetime.now().isocalendar()[1]
active_teams = teams[
    teams['last_activity_week'] == current_week
]
wat = len(active_teams[active_teams['subscription_status'] == 'paid'])

# Activation Rate
new_teams = teams[
    (teams['created_at'] >= datetime.now() - timedelta(days=7)) &
    (teams['subscription_status'] == 'paid')
]
activated = new_teams[new_teams['has_activity'] == True]
activation_rate = len(activated) / len(new_teams) * 100

# TTV
ttv_data = teams[teams['first_doc_creation_date'].notna()]
ttv_seconds = (
    ttv_data['first_doc_creation_date'] -
    ttv_data['created_at']
).mean().total_seconds()
ttv_minutes = ttv_seconds / 60

# 목표 대비 진행률
kpis = {
    'WAT': {'value': wat, 'target': 2500, 'progress': (wat/2500)*100},
    'Activation': {'value': f"{activation_rate:.1f}%", 'target': '60%'},
    'TTV': {'value': f"{ttv_minutes:.0f}분", 'target': '12분'}
}

# 저장
df_kpi = pd.DataFrame([kpis]).T
df_kpi.to_csv(f'kpis_{datetime.now().date()}.csv')

# Slack 알림
message = f"""
📊 일일 KPI 리포트 - {datetime.now().strftime('%Y-%m-%d')}

🎯 WAT: {wat}팀 (목표 2,500 / 진행률 {(wat/2500)*100:.0f}%)
🚀 Activation: {activation_rate:.1f}% (목표 60%)
⚡ TTV: {ttv_minutes:.0f}분 (목표 12분)

대시보드: https://looker.company.com/kpis
"""

requests.post(SLACK_WEBHOOK, json={"text": message})
```

## 3단계: 경보(Alert) 설계

P1 (긴급), P2 (중요), P3 (정보)로 분류:

### P1 경보 (즉시 대응)

```
조건: WAT가 전주 대비 20% 이상 하락
     또는 Activation Rate < 40%

Slack 채널: #kpi-critical
담당자: Head of Growth, Product Manager
반응 시간: 1시간 이내

예시:
🚨 P1 Alert - WAT Sudden Drop

전주: 2,470팀 (-19.3%)

원인 분석 필수!
```

## MCP & n8n 자동화 확장 (Advanced)

### MCP (Model Context Protocol) 연동

Claude Code에서 직접 데이터베이스 쿼리:

```
> 지난 7일간 Daily Active Teams 추이를 그래프로 보여줘.

Claude: 데이터베이스에 쿼리합니다.

@mcp-analytics SELECT
    DATE(timestamp) as date,
    COUNT(DISTINCT team_id) as dat
FROM activities
WHERE timestamp >= NOW() - INTERVAL '7 days'
GROUP BY DATE(timestamp)
ORDER BY date

[라인 차트 표시]
```

## PM 역할 확장 포인트

### BEFORE: 대시보드 설정 = 기술팀 의존
```
PM이 KPI 정의 요청
  ↓
엔지니어링/데이터팀 배정 (1~2주)
  ↓
데이터베이스 쿼리 작성 (3~5일)
  ↓
대시보드 구축 (1주)
  ↓
PM이 정기적으로 수동 확인

문제점:
- KPI 변경 어려움
- 실시간 모니터링 불가
- 기술팀 의존도 높음
```

---

### AFTER: PM 주도의 KPI 관리
```
PM이 Claude와 함께 KPI 정의 (2시간)
  ↓
PM이 경보 기준 설정 (1시간)
  ↓
Claude가 자동화 스크립트 작성 (30분)
  ↓
데이터팀: 매일 자동화된 결과 검토 (30분/주)
  ↓
PM이 매일 자동 알림으로 모니터링

이점:
- 실시간 KPI 추적
- 신속한 대응
- 기술팀 의존도 최소화
- 유연한 KPI 변경 가능
```

**PM의 새로운 책임**:
- KPI 정의 → 지표 소유권 강화
- 경보 기준 설정 → 주도적 모니터링
- 주간 리뷰 → 데이터 기반 의사결정

**시간 절감**: 대시보드 구축 2~3주 → 2시간
**자동화 정도**: 수동 확인 → 자동 알림 시스템
