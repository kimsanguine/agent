# 경쟁사 분석: 구조화된 인텔리전스

## 개요
Discovery 단계에서 **경쟁 환경을 이해하는 것**은 우리의 포지셔닝과 차별화 전략의 기초입니다.

Claude Code는:
- 산재된 경쟁사 자료를 빠르게 구조화
- 멀티소스 데이터를 병렬로 분석해 일관성 있는 인텔리전스 생성
- 선택과 포기(trade-off)가 명확한 비교 분석

을 통해 PM이 **경쟁 전략 수립에 집중**할 수 있도록 지원합니다.

---

## 섹션 1: 기존 자료 기반 비교 분석

### 상황
마케팅, 영업 팀으로부터 수집된 경쟁사 자료가 있습니다:
- 경쟁사별 pricing-page 스크린샷
- G2, Capterra 리뷰 요약
- 뉘앙스가 담긴 피칭 자료들 (산재된 상태)

PM은 이 자료들을 종합해 **우리 제품의 화이트 스페이스**를 찾아야 합니다.

### Claude Code 워크플로우

```bash
cc claude code

> @competitor-data/ 이 폴더의 경쟁사 자료를 분석해서:
  1. 경쟁사별 가격/기능/타겟 고객 비교 매트릭스를 만들어줘
  2. 각 경쟁사의 강점/약점을 정리해줘
  3. 우리(Acme 기준)가 들어갈 수 있는 화이트스페이스 영역을 찾아줘
     (경쟁사는 못 하는데, 고객이 원하는 영역)
```

### Claude의 응답 (실제 터미널 아웃풋)

```
Claude: 3개 경쟁사 자료를 분석했습니다.

## 1. 가격/기능 비교 매트릭스

| 항목 | Acme (우리) | Competitor-A | Competitor-B | Competitor-C |
|------|----------|-------------|-------------|-------------|
| **가격대** | $29-99/mo | $99-299/mo | $49-149/mo | Free-$79/mo |
| **타겟** | SMB/Mid | Enterprise | Mid/Enterprise | Consumer |
| **주 기능** | 통합 관리 | 엔터프라이즈 협업 | 경량 협업 | 개인 생산성 |
| **API 지원** | Yes | Yes | Limited | No |
| **커스터마이징** | 제한적 | 높음 | 중간 | 없음 |
| **한국어** | 지원 | 미지원 | 미지원 | 미지원 |
| **교육 기능** | Yes | No | Yes | Limited |

## 2. 경쟁사별 강점/약점 분석

### Competitor-A (고가 엔터프라이즈)
**강점:**
- 깊은 커스터마이징 가능
- 엔터프라이즈급 보안/규정준수
- 대규모 팀 협업에 최적화
- 전문 컨설턴트 지원

**약점:**
- 고가격 ($99-299/mo)
- 온보딩 복잡 (구성 기간 2-4주)
- SMB가 접근하기 어려움
- 한국어 미지원
- G2 평점: 4.1/5 (엔터프라이즈 사용자 중심)

### Competitor-B (경량 협업 도구)
**강점:**
- 합리적 가격 ($49-149/mo)
- 빠른 도입 (1-2주)
- 직관적 UI

**약점:**
- API 제한적 → 통합 어려움
- 고급 분석 기능 부족
- 대규모 조직에 확장성 제한
- 한국어 미지원
- G2 평점: 3.8/5 (한계 도달 지점 언급 다수)

### Competitor-C (개인 생산성 도구)
**강점:**
- 무료 플랜으로 진입장벽 낮음
- 매우 사용하기 쉬움
- 라이트 사용자 선호도 높음

**약점:**
- 팀 협업 기능 부족
- 엔터프라이즈 기능 거의 없음
- 수익화 모델 약함 (free tier가 너무 좋음)
- G2 평점: 3.9/5 (비즈니스 용도로 부족하다는 평 많음)

## 3. 화이트스페이스: Acme의 기회 영역

### 핵심 발견: "중간 지점의 공백"

**갭 1: SMB-친화적 + 팀 협업 + 합리적 가격**
- Competitor-A: 너무 비싸고 구성 복잡
- Competitor-C: 팀 기능 부족
- Competitor-B: 가격은 OK인데 API/통합 약함
→ **Acme의 위치**: SMB가 $30-50으로 도입 가능한, 팀 협업 + 통합 가능한 도구

**갭 2: 한국어 + 로컬 고객 지원**
- 모든 경쟁사가 한국어 미지원
- 한국 SMB 시장에서 untapped opportunity
→ **Acme의 위치**: 한국 중소기업의 첫 협업 도구

**갭 3: 구현 용이성 + 기능성 균형**
- Competitor-A: 기능은 많은데 너무 복잡
- Competitor-C: 간단한데 기능 부족
- Acme G2 리뷰: "간단하면서도 우리 팀이 원하는 기능이 다 있다"
→ **Acme의 위치**: Goldilocks zone (적당함)

## 4. 위험 신호 (모니터링 필요)

| 위험 신호 | 영향도 | 모니터링 방법 |
|----------|--------|------------|
| Competitor-A의 가격 인하 | 높음 | 분기별 가격 페이지 모니터링 |
| Competitor-B의 API 개선 | 중간 | 분기별 업데이트 로그 확인 |
| Competitor-C의 팀 기능 추가 | 중간 | 월별 제품 로드맵 모니터링 |
| 새로운 경쟁사의 등장 | 높음 | Crunchbase, ProductHunt 모니터링 |

[competitive-analysis-matrix.md 파일이 생성되었습니다]
```

```
## Agent 3: 가격 전략 분석

### 가격 포지셔닝
```
가격 축 →

Competitor-C    Competitor-B    Competitor-A
$0-79/mo        $49-149/mo      $99-299/mo
(Free tier)     (Mid-market)    (Enterprise)

기능성 축
    ↓

Competitor-C         Competitor-B
  (Light)           (Full-featured)     Competitor-A
                                        (Full + Enterprise)
```
### 각사의 수익화 전략

**Competitor-A: "스키밍" 전략**
- 고가 시장에 고가격 제시
- 한계 고객 포기 (SMB)
- Enterprise 최대화

**Competitor-B: "침투" 전략**
- 중간 가격으로 broad market 확보
- 기능에 제약 (고급 기능 별도 요금)
- 빠른 성장 추구

**Competitor-C: "프리미엄" 전략**
- 무료로 진입장벽 제거
- Free → Paid 전환율 낮음 (3-5% 추정)
- CAC 회수에 어려움

### Acme의 가격 전략 평가

**현재 위치: $29-99/mo (Competitor-B와 C 사이)**
- 강점: 가장 저렴함 → SMB에 접근성 높음
- 약점: 기능이 부족하면 "싼 이유"가 될 수 있음
- 전략: "가성비"를 전면에 (기능 + 가격 모두 강조)

## 최종 통합 분석: Acme의 위치

### 시장 포지셔닝 맵
```
기능성
  ↑
  | Competitor-A
  | (엔터프라이즈)
  |
  |     Acme
  |     (가성비)
  |  Competitor-B
  |
  |  Competitor-C
  |  (초경량)
  └─────────────────→ 가격 (낮음)
```
### Acme의 강점 (리뷰/데이터 기반)
1. **가격**: 가장 저렴 → 초기 진입 용이
2. **기능성**: Competitor-C보다 낫고, Competitor-A보다 간단
3. **사용성**: 중간 수준 (Competitor-C 수준의 간편성은 아니지만, Competitor-A 수준으로 복잡하지 않음)

### Acme의 약점
1. **규모 확장성**: 대규모 팀에는 Competitor-A 필요 (고객 이탈)
2. **고객 지원**: G2 리뷰에서 언급 부족 (강조 필요)
3. **마켓 인지도**: Competitor-C (8,923개 리뷰) vs Acme (리뷰 수 미지원)

### 모니터링 신호

🔴 **즉시 대응이 필요한 신호:**
- Competitor-B가 $29 가격대로 인하하는 경우
  → 기능 차별화 전략 필요

🟡 **분기별 모니터링:**
- Competitor-A의 SMB 대상 저가 상품 출시
- Competitor-C의 팀 기능 고도화

[competitive-intel-final.md 파일이 생성되었습니다]
```

### 💡 PM이 주목해야 할 포인트

1. **감정 분석의 함정**
  - App Store 리뷰는 극단주의자 중심 (매우 좋거나 매우 나쁜 사람)
  - 중간 사용자는 리뷰를 작성하지 않음 → 편향된 데이터
  - **검증 필요**: 평균적 고객의 실제 만족도는?

2. **가격 전략의 선택**
  - "가성비"라는 포지셔닝이 실제로 고객에게 통하는가?
  - 저가 전략이 "저품질" 인식으로 연결되지는 않는가?

3. **의사결정**
  - 어느 경쟁사를 주 경쟁사로 볼 것인가?
  - 각 경쟁사별 이기는 전략은?
